# 应用窗口类型详解

## 引言

除了 Activity 的主窗口，Android 应用还可以创建多种类型的窗口，如 Dialog、Toast、PopupWindow、悬浮窗等。这些窗口各有特点和使用场景。本文将详细介绍各种应用窗口类型的原理和实现。

---

## 1. 窗口类型分类

### 1.1 窗口类型层次

```
┌─────────────────────────────────────────────────────────────────────┐
│                         窗口类型层次                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  系统窗口 (System Windows) - TYPE: 2000~2999                        │
│  ├── 状态栏 (TYPE_STATUS_BAR = 2000)                                │
│  ├── 搜索栏 (TYPE_SEARCH_BAR = 2001)                                │
│  ├── Toast (TYPE_TOAST = 2005)                                      │
│  ├── 输入法 (TYPE_INPUT_METHOD = 2011)                              │
│  ├── 导航栏 (TYPE_NAVIGATION_BAR = 2019)                            │
│  └── 应用悬浮窗 (TYPE_APPLICATION_OVERLAY = 2038)                    │
│                                                                     │
│  子窗口 (Sub Windows) - TYPE: 1000~1999                             │
│  ├── 应用面板 (TYPE_APPLICATION_PANEL = 1000)                        │
│  ├── 媒体窗口 (TYPE_APPLICATION_MEDIA = 1001)                        │
│  └── 子面板 (TYPE_APPLICATION_SUB_PANEL = 1002)                      │
│                                                                     │
│  应用窗口 (Application Windows) - TYPE: 1~99                        │
│  ├── 基础应用窗口 (TYPE_BASE_APPLICATION = 1)                        │
│  ├── 普通应用窗口 (TYPE_APPLICATION = 2)                             │
│  └── 启动窗口 (TYPE_APPLICATION_STARTING = 3)                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

Z-Order: 系统窗口 > 子窗口 > 应用窗口
```

### 1.2 应用可创建的窗口类型

| 窗口类型 | 说明 | 是否需要权限 |
|---------|------|------------|
| Activity 窗口 | 应用主窗口 | 否 |
| Dialog | 对话框 | 否 |
| PopupWindow | 弹出窗口 | 否 |
| Toast | 消息提示 | 否 |
| 悬浮窗 | 覆盖其他应用的窗口 | 是 (SYSTEM_ALERT_WINDOW) |

## 2. Dialog 详解

### 2.1 Dialog 与 Activity 窗口的关系

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Activity + Dialog 窗口关系                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  WMS 视角：                                                          │
│  ┌─────────────────────────────────────────────┐                   │
│  │ WindowToken (Activity Token)                │                   │
│  │   ├── WindowState (Activity 窗口)            │ ◄── Z-Order 低   │
│  │   └── WindowState (Dialog 窗口)              │ ◄── Z-Order 高   │
│  └─────────────────────────────────────────────┘                   │
│                                                                     │
│  同一个 Token 下的窗口共享相同的生命周期                               │
│  Dialog 销毁时，其 WindowState 也会被移除                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 Dialog 核心实现

```java
// Dialog.java
public class Dialog implements DialogInterface, Window.Callback, ... {
    
    // Dialog 有自己的 Window
    private final Window mWindow;
    
    // 但使用 Activity 的 WindowManager
    private WindowManager mWindowManager;
    
    // 使用 Activity 的 Context
    final Context mContext;
    
    public Dialog(Context context) {
        // 1. 创建 PhoneWindow
        mWindow = new PhoneWindow(mContext);
        
        // 2. 获取 WindowManager（使用 Activity 的）
        mWindowManager = (WindowManager) context.getSystemService(Context.WINDOW_SERVICE);
        
        // 3. 设置回调
        mWindow.setCallback(this);
    }
    
    public void show() {
        // 1. 分发 onStart
        onStart();
        
        // 2. 获取 DecorView
        mDecor = mWindow.getDecorView();
        
        // 3. 设置窗口属性
        WindowManager.LayoutParams l = mWindow.getAttributes();
        l.token = mContext.getActivityToken();  // 重要：使用 Activity 的 token
        
        // 4. 添加到 WindowManager
        mWindowManager.addView(mDecor, l);
        
        mShowing = true;
    }
    
    public void dismiss() {
        // 从 WindowManager 移除
        mWindowManager.removeViewImmediate(mDecor);
        mShowing = false;
    }
}
```

## 3. PopupWindow 详解

### 3.1 PopupWindow 原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                      PopupWindow 结构                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  PopupWindow                                                        │
│    │                                                                │
│    ├── mContentView (用户设置的内容视图)                              │
│    │                                                                │
│    ├── mDecorView (PopupDecorView，包装内容视图)                      │
│    │     └── PopupBackgroundView                                    │
│    │           └── mContentView                                     │
│    │                                                                │
│    └── WindowManager.LayoutParams                                   │
│          ├── type = TYPE_APPLICATION_PANEL (子窗口)                  │
│          ├── token = 锚点 View 的窗口 token                          │
│          └── flags = FLAG_NOT_TOUCH_MODAL 等                        │
│                                                                     │
│  特点：                                                              │
│  • 不需要创建 PhoneWindow                                            │
│  • 直接将 View 添加到 WindowManager                                  │
│  • 窗口类型是子窗口 (Sub-Window)                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 PopupWindow 核心实现

```java
// PopupWindow.java
public class PopupWindow {
    // 内容视图
    private View mContentView;
    
    // 装饰视图
    private PopupDecorView mDecorView;
    
    // WindowManager
    private WindowManager mWindowManager;
    
    // 窗口属性
    private final WindowManager.LayoutParams mWindowLayoutParams;
    
    public PopupWindow(View contentView, int width, int height) {
        mContentView = contentView;
        mWidth = width;
        mHeight = height;
    }
    
    public void showAtLocation(View parent, int gravity, int x, int y) {
        // 1. 获取 WindowManager
        mWindowManager = (WindowManager) mContext.getSystemService(Context.WINDOW_SERVICE);
        
        // 2. 创建装饰视图
        mDecorView = createDecorView(mContentView);
        
        // 3. 准备窗口参数
        WindowManager.LayoutParams p = createPopupLayoutParams(
                parent.getWindowToken());  // 使用 parent 的 token
        p.gravity = gravity;
        p.x = x;
        p.y = y;
        
        // 4. 添加到 WindowManager
        mWindowManager.addView(mDecorView, p);
    }
    
    public void showAsDropDown(View anchor, int xoff, int yoff, int gravity) {
        // 1. 计算位置（相对于 anchor）
        findDropDownPosition(anchor, mWindowLayoutParams, xoff, yoff, gravity);
        
        // 2. 准备参数
        WindowManager.LayoutParams p = mWindowLayoutParams;
        p.token = anchor.getWindowToken();  // 使用锚点的 token
        
        // 3. 添加窗口
        mWindowManager.addView(mDecorView, p);
        
        mIsShowing = true;
        mIsDropdown = true;
    }
    
    private WindowManager.LayoutParams createPopupLayoutParams(IBinder token) {
        WindowManager.LayoutParams p = new WindowManager.LayoutParams();
        
        // 子窗口类型
        p.type = WindowManager.LayoutParams.TYPE_APPLICATION_PANEL;
        
        // 使用父窗口的 token
        p.token = token;
        
        // 窗口标志
        p.flags = computeFlags(p.flags);
        
        return p;
    }
    
    public void dismiss() {
        if (mIsShowing && mDecorView != null) {
            mWindowManager.removeViewImmediate(mDecorView);
            mIsShowing = false;
        }
    }
}
```

### 3.3 PopupWindow 与 Dialog 对比

| 特性 | Dialog | PopupWindow |
|-----|--------|-------------|
| 窗口类型 | 应用窗口 | 子窗口 |
| 是否有 Window | 是 (PhoneWindow) | 否 |
| 位置控制 | 通常居中 | 精确控制 |
| 背景调暗 | 支持 | 不支持 |
| 使用场景 | 用户交互 | 下拉菜单、提示 |
| 焦点处理 | 可获取焦点 | 可配置 |

## 4. Toast 详解

### 4.1 Toast 窗口机制

```
┌─────────────────────────────────────────────────────────────────────┐
│                         Toast 显示流程                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  应用进程                              System Server                 │
│  ┌────────────────┐                   ┌──────────────────────┐     │
│  │ Toast.show()   │                   │ NotificationManager  │     │
│  │      │         │                   │   Service (NMS)      │     │
│  │      ▼         │                   │        │             │     │
│  │  getService()  │──── Binder ───────│        ▼             │     │
│  │  enqueueToast()│                   │  ToastQueue.add()    │     │
│  └────────────────┘                   │        │             │     │
│                                       │        ▼             │     │
│  ┌────────────────┐                   │  调度显示时机         │     │
│  │ TN (Callback)  │◄─── Binder ──────│  TN.show()           │     │
│  │      │         │                   └──────────────────────┘     │
│  │      ▼         │                                                │
│  │  handleShow()  │                                                │
│  │      │         │                                                │
│  │      ▼         │                                                │
│  │ WindowManager  │                                                │
│  │  .addView()    │                                                │
│  └────────────────┘                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 Toast 核心实现

```java
// Toast.java
public class Toast {
    // 内容视图
    View mNextView;
    
    // 显示时长
    int mDuration;
    
    // 回调（TN = TransientNotification）
    final TN mTN;
    
    public static Toast makeText(Context context, CharSequence text, int duration) {
        Toast result = new Toast(context);
        
        // 创建默认布局
        View v = LayoutInflater.from(context).inflate(
            com.android.internal.R.layout.transient_notification, null);
        TextView tv = v.findViewById(com.android.internal.R.id.message);
        tv.setText(text);
        
        result.mNextView = v;
        result.mDuration = duration;
        
        return result;
    }
    
    public void show() {
        INotificationManager service = getService();
        String pkg = mContext.getOpPackageName();
        TN tn = mTN;
        tn.mNextView = mNextView;
        
        // 通过 NMS 排队显示
        service.enqueueToast(pkg, mToken, tn, mDuration, displayId);
    }
    
    // TN 是 Binder 回调，NMS 通过它控制 Toast 显示/隐藏
    private static class TN extends ITransientNotification.Stub {
        View mView;
        View mNextView;
        WindowManager mWM;
        
        final WindowManager.LayoutParams mParams = new WindowManager.LayoutParams();
        
        TN(String packageName) {
            // 窗口类型：Toast
            mParams.type = WindowManager.LayoutParams.TYPE_TOAST;
            
            // 窗口标志：不可获取焦点、不可触摸
            mParams.flags = WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE
                    | WindowManager.LayoutParams.FLAG_NOT_TOUCHABLE;
        }
        
        @Override
        public void show(IBinder windowToken) {
            mHandler.obtainMessage(SHOW, windowToken).sendToTarget();
        }
        
        public void handleShow(IBinder windowToken) {
            // 设置 token
            mParams.token = windowToken;
            
            if (mView != mNextView) {
                // 移除旧的
                handleHide();
                mView = mNextView;
            }
            
            // 添加到 WindowManager
            mWM.addView(mView, mParams);
        }
        
        @Override
        public void hide() {
            mHandler.obtainMessage(HIDE).sendToTarget();
        }
        
        public void handleHide() {
            if (mView != null) {
                mWM.removeViewImmediate(mView);
                mView = null;
            }
        }
    }
}
```

### 4.3 Toast 显示策略

```java
// NotificationManagerService.java 中的 Toast 管理
class ToastRecord {
    final String pkg;
    final ITransientNotification callback;
    int duration;
    IBinder token;
}

// Toast 队列
final ArrayList<ToastRecord> mToastQueue = new ArrayList<>();

void enqueueToast(String pkg, ITransientNotification callback, int duration) {
    // 1. 检查权限（Android 11+ 后台应用受限）
    // 2. 限制每个应用同时显示的 Toast 数量
    // 3. 添加到队列
    mToastQueue.add(new ToastRecord(...));
    
    // 4. 如果当前没有显示，开始显示
    if (mToastQueue.size() == 1) {
        showNextToastLocked();
    }
}

void showNextToastLocked() {
    ToastRecord record = mToastQueue.get(0);
    record.callback.show(record.token);
    
    // 设置超时自动取消
    scheduleTimeoutLocked(record);
}
```

### 5.2 悬浮窗实现

```java
// 悬浮窗服务示例
public class FloatingWindowService extends Service {
    
    private WindowManager windowManager;
    private View floatingView;
    
    @Override
    public void onCreate() {
        super.onCreate();
        
        // 1. 获取 WindowManager
        windowManager = (WindowManager) getSystemService(WINDOW_SERVICE);
        
        // 2. 创建悬浮窗视图
        floatingView = LayoutInflater.from(this)
            .inflate(R.layout.floating_layout, null);
        
        // 3. 配置窗口参数
        WindowManager.LayoutParams params = new WindowManager.LayoutParams(
            WindowManager.LayoutParams.WRAP_CONTENT,
            WindowManager.LayoutParams.WRAP_CONTENT,
            // Android 8.0+ 使用 TYPE_APPLICATION_OVERLAY
            Build.VERSION.SDK_INT >= Build.VERSION_CODES.O
                ? WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY
                : WindowManager.LayoutParams.TYPE_PHONE,
            // 不获取焦点，不遮挡输入
            WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE,
            PixelFormat.TRANSLUCENT
        );
        
        // 4. 设置初始位置
        params.gravity = Gravity.TOP | Gravity.START;
        params.x = 0;
        params.y = 100;
        
        // 5. 添加到 WindowManager
        windowManager.addView(floatingView, params);
        
        // 6. 设置拖动监听
        setupTouchListener(params);
    }
    
    private void setupTouchListener(WindowManager.LayoutParams params) {
        floatingView.setOnTouchListener(new View.OnTouchListener() {
            private int lastX, lastY;
            private int initialX, initialY;
            
            @Override
            public boolean onTouch(View v, MotionEvent event) {
                switch (event.getAction()) {
                    case MotionEvent.ACTION_DOWN:
                        initialX = params.x;
                        initialY = params.y;
                        lastX = (int) event.getRawX();
                        lastY = (int) event.getRawY();
                        return true;
                        
                    case MotionEvent.ACTION_MOVE:
                        // 计算位移
                        int dx = (int) event.getRawX() - lastX;
                        int dy = (int) event.getRawY() - lastY;
                        params.x = initialX + dx;
                        params.y = initialY + dy;
                        // 更新位置
                        windowManager.updateViewLayout(floatingView, params);
                        return true;
                }
                return false;
            }
        });
    }
    
    @Override
    public void onDestroy() {
        super.onDestroy();
        if (floatingView != null) {
            windowManager.removeView(floatingView);
        }
    }
}
```

### 5.3 悬浮窗权限请求

```java
// 检查和请求悬浮窗权限
public class FloatingPermissionHelper {
    
    public static boolean canDrawOverlays(Context context) {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
            return Settings.canDrawOverlays(context);
        }
        return true;  // Android 6.0 以下默认允许
    }
    
    public static void requestPermission(Activity activity) {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
            Intent intent = new Intent(
                Settings.ACTION_MANAGE_OVERLAY_PERMISSION,
                Uri.parse("package:" + activity.getPackageName())
            );
            activity.startActivityForResult(intent, REQUEST_CODE_OVERLAY);
        }
    }
    
    // 在 onActivityResult 中检查结果
    public static void onActivityResult(int requestCode, Activity activity) {
        if (requestCode == REQUEST_CODE_OVERLAY) {
            if (canDrawOverlays(activity)) {
                // 权限已授予，可以显示悬浮窗
            } else {
                // 权限被拒绝
            }
        }
    }
}
```

## 7. 窗口类型选择指南

### 7.1 场景对照表

| 场景 | 推荐类型 | 原因 |
|-----|---------|------|
| 用户确认/选择 | AlertDialog | 标准交互模式 |
| 下拉菜单 | PopupWindow | 精确定位 |
| 操作提示 | Toast | 简单、自动消失 |
| 悬浮工具 | TYPE_APPLICATION_OVERLAY | 跨应用显示 |
| 自定义弹窗 | Dialog/PopupWindow | 取决于是否需要模态 |
| 全局消息 | 通知栏 | 符合系统规范 |

### 7.2 窗口创建检查清单

```
创建窗口前的检查：

□ 确定窗口类型
  - 需要模态效果？ → Dialog
  - 需要精确定位？ → PopupWindow
  - 需要跨应用显示？ → 悬浮窗

□ 检查权限
  - 悬浮窗需要 SYSTEM_ALERT_WINDOW 权限
  - Android 10+ 后台启动 Activity 受限

□ 考虑生命周期
  - 依附于 Activity？ 需处理 Activity 销毁
  - 依附于 Service？ 需处理 Service 停止

□ 处理配置变化
  - 屏幕旋转
  - 分屏模式切换
```

## 8. 本章小结

本文介绍了各种应用窗口类型：

1. **Dialog**：有 PhoneWindow，使用 Activity Token，适合用户交互
2. **PopupWindow**：无 Window，子窗口类型，适合精确定位
3. **Toast**：系统管理，队列显示，适合简单提示
4. **悬浮窗**：TYPE_APPLICATION_OVERLAY，需要权限，跨应用显示
