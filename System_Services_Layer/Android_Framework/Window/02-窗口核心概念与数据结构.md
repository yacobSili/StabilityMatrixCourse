# 窗口核心概念与数据结构

## 引言

在深入窗口系统之前，必须先理解几个核心概念：Window、Surface、Layer 以及它们之间的关系。这些概念贯穿整个窗口系统，是理解后续章节的基础。

本文将详细介绍窗口系统中的核心概念和关键数据结构，帮助读者建立清晰的心智模型。

---

## 1. 核心概念辨析

### 1.1 Window、Surface、Layer 的关系

这三个概念是窗口系统中最容易混淆的，它们分别存在于不同层级：

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用层视角                                │
│                                                                 │
│    Window (窗口)                                                │
│    ├── 逻辑概念，代表一个可显示的矩形区域                         │
│    ├── 在应用中体现为 PhoneWindow                                │
│    └── 管理 DecorView（View 树的根）                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │ 1:1 关联
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                       Framework 层视角                           │
│                                                                 │
│    Surface (绘图表面)                                            │
│    ├── 窗口的绘图缓冲区抽象                                       │
│    ├── 应用通过 Surface 获取 Canvas 进行绘制                      │
│    └── 底层关联 BufferQueue                                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │ 1:1 关联
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      SurfaceFlinger 层视角                       │
│                                                                 │
│    Layer (图层)                                                  │
│    ├── SF 中可合成的图形单元                                      │
│    ├── 持有 BufferQueue，接收应用提交的帧                         │
│    └── 参与最终的图形合成                                         │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 概念对照表

| 概念 | 所在层级 | 职责 | 生命周期管理者 |
|-----|---------|------|---------------|
| Window | 应用层 | UI 容器，管理 View 树 | Activity/Dialog |
| PhoneWindow | 应用层 | Window 的具体实现 | Activity |
| WindowState | WMS | 窗口在服务端的表示 | WMS |
| Surface | 应用/Framework | 绘图表面抽象 | ViewRootImpl |
| SurfaceControl | Framework | Surface 的控制句柄 | WMS |
| Layer | SurfaceFlinger | 可合成的图形层 | SurfaceFlinger |
| BufferQueue | Native | 帧缓冲队列 | Layer |

## 2. 应用层数据结构

### 2.1 Window

`Window` 是一个抽象类，定义了窗口的基本接口：

```java
// frameworks/base/core/java/android/view/Window.java
public abstract class Window {
    // 窗口属性
    private WindowManager.LayoutParams mWindowAttributes;
    
    // 窗口回调
    private Callback mCallback;
    
    // 核心方法
    public abstract void setContentView(int layoutResID);
    public abstract void setContentView(View view);
    public abstract View getDecorView();
    
    // 获取窗口属性
    public final WindowManager.LayoutParams getAttributes() {
        return mWindowAttributes;
    }
}
```

### 2.2 PhoneWindow

`PhoneWindow` 是 `Window` 的唯一实现：

```java
// frameworks/base/core/java/com/android/internal/policy/PhoneWindow.java
public class PhoneWindow extends Window {
    // 装饰视图（窗口的根 View）
    private DecorView mDecor;
    
    // 内容父容器
    private ViewGroup mContentParent;
    
    @Override
    public void setContentView(int layoutResID) {
        if (mContentParent == null) {
            // 1. 安装装饰视图
            installDecor();
        }
        // 2. 加载布局到内容区域
        mLayoutInflater.inflate(layoutResID, mContentParent);
    }
    
    private void installDecor() {
        // 创建 DecorView
        mDecor = generateDecor(-1);
        // 创建内容容器
        mContentParent = generateLayout(mDecor);
    }
}
```

### 2.4 ViewRootImpl

`ViewRootImpl` 是 View 系统与窗口系统的桥梁：

```java
// frameworks/base/core/java/android/view/ViewRootImpl.java
public final class ViewRootImpl implements ViewParent {
    // View 树的根
    View mView;
    
    // 绘图表面
    final Surface mSurface = new Surface();
    
    // 与 WMS 的会话
    final IWindowSession mWindowSession;
    
    // 窗口在 WMS 中的引用
    final W mWindow;  // IWindow.Stub 实现
    
    // Surface 控制
    private final SurfaceControl mSurfaceControl;
    
    // 添加窗口到 WMS
    public void setView(View view, WindowManager.LayoutParams attrs, ...) {
        mView = view;
        // 请求布局
        requestLayout();
        // 添加到 WMS
        res = mWindowSession.addToDisplayAsUser(mWindow, ...);
    }
    
    // 执行遍历（测量、布局、绘制）
    void doTraversal() {
        performTraversals();
    }
    
    private void performTraversals() {
        // 1. 测量
        performMeasure(childWidthMeasureSpec, childHeightMeasureSpec);
        // 2. 布局
        performLayout(lp, mWidth, mHeight);
        // 3. 绘制
        performDraw();
    }
}
```

## 3. Framework 层数据结构 (WMS)

### 3.1 WindowState

`WindowState` 是窗口在 WMS 中的表示：

```java
// frameworks/base/services/core/java/com/android/server/wm/WindowState.java
class WindowState extends WindowContainer<WindowState> {
    // 所属会话
    final Session mSession;
    
    // 客户端窗口引用
    final IWindow mClient;
    
    // 窗口属性
    final WindowManager.LayoutParams mAttrs;
    
    // 窗口令牌
    WindowToken mToken;
    
    // Surface 控制器
    WindowSurfaceController mWinAnimator;
    
    // 窗口帧（位置和大小）
    final Rect mWindowFrames = new Rect();
    
    // 请求的大小
    int mRequestedWidth, mRequestedHeight;
    
    // 是否可见
    boolean mHasSurface;
    
    // 焦点状态
    boolean mFocused;
    
    // 创建 SurfaceControl
    void createSurfaceControl(SurfaceControl.Builder b) {
        mSurfaceControl = b.setName(mWinAnimator.mSurfaceTitle)
                           .setContainerLayer()
                           .build();
    }
}
```

### 3.2 WindowToken

## 4. Native 层数据结构

### 4.1 Surface (Native)

Native 层的 Surface 是应用绘制的直接对象：

```cpp
// frameworks/native/libs/gui/include/gui/Surface.h
class Surface : public ANativeObjectBase<ANativeWindow, Surface, RefBase> {
public:
    // 关联的 BufferQueue 生产者
    sp<IGraphicBufferProducer> mGraphicBufferProducer;
    
    // 缓冲区槽
    BufferSlot mSlots[NUM_BUFFER_SLOTS];
    
    // 出队一个缓冲区用于绘制
    int dequeueBuffer(ANativeWindowBuffer** buffer, int* fenceFd);
    
    // 入队一个缓冲区（绘制完成）

    // 取消缓冲区

};
```

### 4.4 BufferQueue

`BufferQueue` 是生产者-消费者模型的核心：

```cpp
// frameworks/native/libs/gui/include/gui/BufferQueue.h

// 缓冲区槽状态
enum BufferState {
    FREE,           // 空闲，可被出队
    DEQUEUED,       // 已出队，应用正在绘制
    QUEUED,         // 已入队，等待消费
    ACQUIRED,       // 已获取，SF 正在使用
};

// BufferQueue 核心流程
// 
// 应用 (Producer)                    SF (Consumer)
//      │                                  │
//      │ dequeueBuffer()                  │
//      │────────────────►                 │
//      │   返回空闲 Buffer                 │
//      │                                  │
//      │   绘制到 Buffer                   │
//      │                                  │
//      │ queueBuffer()                    │
//      │────────────────►                 │
//      │   Buffer 入队                    │
//      │                                  │
//      │                   acquireBuffer()│
//      │                  ◄────────────────
//      │                   获取已入队 Buffer
//      │                                  │
//      │                   合成到屏幕      │
//      │                                  │
//      │                   releaseBuffer()│
//      │                  ◄────────────────
//      │                   释放 Buffer     │
//      │                                  │

class BufferQueue {
public:
    // 缓冲区槽数量
    static constexpr int NUM_BUFFER_SLOTS = 64;
    
    // 创建生产者和消费者
    static void createBufferQueue(
        sp<IGraphicBufferProducer>* outProducer,
        sp<IGraphicBufferConsumer>* outConsumer);
};
```

### 4.5 GraphicBuffer

`GraphicBuffer` 是实际的图形缓冲区：

```cpp
// frameworks/native/libs/ui/include/ui/GraphicBuffer.h
class GraphicBuffer : public ANativeObjectBase<
        ANativeWindowBuffer, GraphicBuffer, RefBase> {
public:
    // 缓冲区属性
    uint32_t width;
    uint32_t height;
    PixelFormat format;
    uint32_t usage;
    
    // Native 句柄（用于跨进程共享）
    native_handle_t* handle;
    
    // 锁定缓冲区（CPU 访问）
    status_t lock(uint32_t usage, void** vaddr);
    status_t unlock();
    
    // 锁定缓冲区（带 Fence）
    status_t lockAsync(uint32_t usage, void** vaddr, int fenceFd);
    status_t unlockAsync(int* fenceFd);
};
```
