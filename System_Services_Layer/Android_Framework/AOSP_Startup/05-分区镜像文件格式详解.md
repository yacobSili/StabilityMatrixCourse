# 分区镜像文件格式详解

## 学习目标

- 理解 boot.img 的格式和结构
- 掌握 vendor_boot.img 的格式
- 理解文件系统镜像（system.img, vendor.img）的格式
- 了解 vbmeta.img 的格式
- 掌握镜像文件的打包和拆解工具

## 背景介绍

Android 设备使用各种镜像文件来存储分区内容。理解这些镜像文件的格式，有助于理解刷机过程、分析系统问题，以及进行系统定制。从 Android 12 开始，启动镜像的格式发生了变化，引入了 vendor_boot 分区。

## 核心概念

### boot.img 格式

#### Android Boot Image 结构

**格式版本**：
- v0（传统格式）
- v1（Android 9+）
- v2（Android 10+）
- v3（Android 11+）
- v4（Android 12+）

#### Boot Image Header

**结构**（v3/v4）：
```
struct boot_img_hdr_v3 {
    uint8_t magic[BOOT_MAGIC_SIZE];      // "ANDROID!"
    uint32_t kernel_size;                // 内核大小
    uint32_t kernel_addr;                // 内核加载地址
    uint32_t ramdisk_size;               // Ramdisk 大小
    uint32_t ramdisk_addr;                // Ramdisk 加载地址
    uint32_t second_size;                 // 第二引导程序大小（已废弃）
    uint32_t second_addr;                 // 第二引导程序地址（已废弃）
    uint32_t tags_addr;                   // 标签地址
    uint32_t page_size;                   // 页大小（通常 4096）
    uint32_t header_version;              // Header 版本
    uint32_t os_version;                  // OS 版本
    uint8_t name[BOOT_NAME_SIZE];         // 名称
    uint8_t cmdline[BOOT_ARGS_SIZE];      // 内核命令行
    uint8_t id[8];                        // ID
    uint8_t extra_cmdline[BOOT_EXTRA_ARGS_SIZE]; // 额外命令行
    uint32_t recovery_dtbo_size;         // Recovery DTBO 大小
    uint64_t recovery_dtbo_offset;       // Recovery DTBO 偏移
    uint32_t header_size;                 // Header 大小
    uint32_t dtb_size;                    // DTB 大小
    uint64_t dtb_addr;                    // DTB 地址
}
```

#### Boot Image 布局

```
+-------------------+
| Boot Image Header |
+-------------------+
| Kernel            |
+-------------------+
| Ramdisk           |
+-------------------+
| DTB (v2+)         |
+-------------------+
```

#### 内容详解

**1. Kernel（内核）**
- 压缩的 Linux 内核镜像
- 格式：Image（ARM64）或 zImage（ARM）
- 由 Bootloader 加载到内存

**2. Ramdisk**
- 初始根文件系统
- 包含 init 进程和早期启动文件
- 挂载为根文件系统（传统方式）
- System-as-root 下内容合并到 system 分区

**3. Device Tree Blob (DTB)**
- 设备树二进制文件
- 描述硬件配置
- v2+ 格式包含在 boot.img 中

### vendor_boot.img 格式（Android 12+）

#### 设计目的

**分离厂商代码**：
- 将厂商特定的启动组件从 boot.img 分离
- 支持 GKI 架构
- 厂商可以独立更新启动组件

#### Vendor Boot Image Header

**结构**：
```
struct vendor_boot_img_hdr_v3 {
    uint8_t magic[VENDOR_BOOT_MAGIC_SIZE]; // "VNDRBOOT"
    uint32_t header_version;               // Header 版本
    uint32_t page_size;                    // 页大小
    uint32_t kernel_addr;                  // 内核地址（已废弃）
    uint32_t ramdisk_addr;                 // Ramdisk 地址
    uint32_t ramdisk_size;                 // Ramdisk 大小
    uint8_t cmdline[VENDOR_BOOT_ARGS_SIZE]; // 命令行
    uint8_t name[VENDOR_BOOT_NAME_SIZE];    // 名称
    uint32_t header_size;                  // Header 大小
    uint32_t dtb_size;                     // DTB 大小
    uint64_t dtb_addr;                     // DTB 地址
}
```

#### Vendor Boot Image 布局

```
+---------------------------+
| Vendor Boot Image Header  |
+---------------------------+
| Vendor Ramdisk            |
+---------------------------+
| DTB / DTBO                |
+---------------------------+
| Vendor Modules            |
+---------------------------+
```

#### 内容详解

**1. Vendor Ramdisk**
- 厂商特定的 ramdisk
- 包含厂商的 init 脚本和工具

**2. DTB / DTBO**
- 设备树覆盖（Device Tree Overlay）
- 厂商特定的设备树修改

**3. Vendor Modules**
- 厂商内核模块（.ko 文件）
- GKI 架构下的动态加载模块

### 文件系统镜像格式

#### system.img / vendor.img 等

**格式类型**：

**1. Raw Ext4 镜像**
- 完整的 ext4 文件系统镜像
- 可以直接挂载
- 文件较大

**2. Sparse 镜像**
- 压缩格式，只包含非零块
- 文件较小
- 需要转换才能挂载

**3. EROFS 镜像（Android 13+）**
- 只读文件系统
- 更高的压缩比
- 更快的挂载速度

#### Sparse 镜像格式

**结构**：
```
+-------------------+
| Sparse Header     |
+-------------------+
| Chunk 1           |
+-------------------+
| Chunk 2           |
+-------------------+
| ...               |
+-------------------+
```

**Chunk 类型**：
- `CHUNK_TYPE_RAW` - 原始数据
- `CHUNK_TYPE_FILL` - 填充数据
- `CHUNK_TYPE_DONT_CARE` - 不需要的数据
- `CHUNK_TYPE_CRC32` - CRC32 校验

### vbmeta.img 格式

#### Android Verified Boot

**作用**：存储分区验证信息

#### VBMeta 结构

**主要内容**：
- 分区描述符
- 哈希树根
- 签名信息
- 验证链信息

**格式**：
```
struct AvbVBMetaImageHeader {
    uint8_t magic[4];           // "AVB0"
    uint32_t required_libavb_version_major;
    uint32_t required_libavb_version_minor;
    uint64_t authentication_data_block_size;
    uint64_t auxiliary_data_block_size;
    uint32_t algorithm_type;
    uint64_t hash_offset;
    uint64_t hash_size;
    uint64_t signature_offset;
    uint64_t signature_size;
    uint64_t public_key_offset;
    uint64_t public_key_size;
    uint64_t public_key_metadata_offset;
    uint64_t public_key_metadata_size;
    uint64_t descriptors_offset;
    uint64_t descriptors_size;
    uint64_t rollback_index;
    uint32_t flags;
    uint32_t rollback_index_location;
    uint8_t release_string[AVB_RELEASE_STRING_SIZE];
    uint8_t reserved[80];
}
```

### 镜像文件工具

#### mkbootimg / unpack_bootimg

**打包 boot.img**：
```bash
mkbootimg \
    --kernel Image \
    --ramdisk ramdisk.img \
    --dtb dtb.img \
    --output boot.img
```

**拆解 boot.img**：
```bash
unpack_bootimg \
    --boot_img boot.img \
    --out output_dir/
```

#### simg2img / img2simg

**Sparse 转 Raw**：
```bash
simg2img system.img system_raw.img
```

**Raw 转 Sparse**：
```bash
img2simg system_raw.img system.img
```

#### avbtool

**生成 vbmeta**：
```bash
avbtool make_vbmeta_image \
    --output vbmeta.img \
    --key key.pem \
    --algorithm SHA256_RSA4096 \
    --prop com.android.build.system.fingerprint:$(BUILD_FINGERPRINT)
```

**验证 vbmeta**：
```bash
avbtool verify_image --image vbmeta.img
```

#### 其他工具

**file 命令**：
```bash
# 查看镜像类型
file boot.img
file system.img
```

**hexdump**：
```bash
# 查看镜像头部
hexdump -C boot.img | head -20
```

## 实际应用

### 拆解 boot.img

```bash
# 使用 unpack_bootimg
unpack_bootimg --boot_img boot.img --out boot_unpacked/

# 查看内容
ls -lh boot_unpacked/
```

### 查看文件系统镜像

```bash
# 转换 Sparse 镜像
simg2img system.img system_raw.img

# 挂载镜像
sudo mount -o loop system_raw.img /mnt

# 查看内容
ls -la /mnt/

# 卸载
sudo umount /mnt
```

### 分析镜像内容

```bash
# 查看 boot.img 信息
file boot.img
hexdump -C boot.img | head -50

# 查看 system.img 信息
file system.img
```

## 总结

### 核心要点

1. **boot.img** 包含内核、ramdisk 和 DTB
2. **vendor_boot.img** 分离厂商启动组件
3. **文件系统镜像** 可以是 Raw、Sparse 或 EROFS 格式
4. **vbmeta.img** 存储验证启动信息
5. **工具** 用于打包、拆解和转换镜像

### 关键格式

- **Boot Image**：内核 + Ramdisk + DTB
- **Sparse Image**：压缩的文件系统镜像
- **VBMeta**：验证启动元数据
