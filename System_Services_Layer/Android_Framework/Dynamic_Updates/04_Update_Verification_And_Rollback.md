## 1. AVB验证机制

Android Verified Boot (AVB) 用于验证分区完整性：

### 1.1 验证流程

1. Bootloader验证vbmeta
2. vbmeta验证各个分区
3. 分区验证通过后加载

### 1.2 验证链

```
vbmeta (根)
├── system (通过vbmeta_system验证)
├── vendor (通过vbmeta_vendor验证)
└── boot (直接验证)
```

## 2. 签名验证

### 2.1 OTA包签名

OTA包必须使用私钥签名，设备使用公钥验证。

### 2.2 验证过程

1. 下载OTA包
2. 验证OTA包签名
3. 验证通过后应用更新

## 3. 更新失败处理

### 3.1 失败检测

- 启动失败检测
- 验证失败检测
- 应用失败检测

### 3.2 失败处理

1. 记录失败信息
2. 自动回滚到原槽位
3. 报告失败原因

---

## 4. 回滚机制

### 4.1 自动回滚

如果新系统无法启动，自动切换回原槽位。

### 4.2 手动回滚

```bash
# 切换槽位
setprop ro.boot.slot_suffix _a
reboot
```

更新验证和回滚要点：

3. **失败处理**：自动检测和处理失败
4. **回滚机制**：快速恢复到原系统
