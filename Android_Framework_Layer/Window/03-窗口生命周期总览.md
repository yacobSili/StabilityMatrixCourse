# 窗口生命周期总览

## 引言

窗口从创建到显示再到销毁，经历了一个完整的生命周期。理解这个生命周期，是掌握窗口系统的关键。本文将从全局视角介绍窗口生命周期的各个阶段，以及各层组件在每个阶段的协作方式。

---

## 1. 窗口生命周期概述

### 1.1 生命周期阶段

```
┌─────────────────────────────────────────────────────────────────┐
│                      窗口生命周期                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    创建阶段          显示阶段          更新阶段        销毁阶段  │
│   ┌───────┐        ┌───────┐        ┌───────┐       ┌───────┐  │
│   │       │        │       │        │       │       │       │  │
│   │ Create│───────►│ Show  │───────►│Update │──────►│Destroy│  │
│   │       │        │       │◄───────│       │       │       │  │
│   └───────┘        └───────┘        └───────┘       └───────┘  │
│                                                                 │
│   • addWindow      • 首次绘制       • 布局变化     • removeView │
│   • 创建 Surface   • 显示到屏幕     • 内容更新     • 释放资源   │
│   • 创建 Layer     • 获取焦点       • 动画效果     • 销毁 Layer │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 各阶段核心操作

| 阶段 | 应用层 | WMS | SurfaceFlinger |
|-----|-------|-----|----------------|
| 创建 | addView | addWindow, 创建 WindowState | 创建 Layer |
| 显示 | performTraversals | relayoutWindow | 首帧合成 |
| 更新 | invalidate/requestLayout | 布局计算 | 帧更新 |
| 销毁 | removeView | removeWindow | 销毁 Layer |

### 2.2 关键代码路径

**应用层 - WindowManager.addView()**

```java
// WindowManagerImpl.java
public void addView(View view, ViewGroup.LayoutParams params) {
    mGlobal.addView(view, params, mContext.getDisplayNoVerify(), mParentWindow, ...);
}

// WindowManagerGlobal.java
public void addView(View view, ViewGroup.LayoutParams params, ...) {
    // 1. 创建 ViewRootImpl
    ViewRootImpl root = new ViewRootImpl(view.getContext(), display);
    
    // 2. 保存到列表
    mViews.add(view);
    mRoots.add(root);
    mParams.add(wparams);
    
    // 3. 关联 View 和 ViewRootImpl
    root.setView(view, wparams, panelParentView, userId);
}
```

**ViewRootImpl.setView()**

```java
// ViewRootImpl.java
public void setView(View view, WindowManager.LayoutParams attrs, ...) {
    synchronized (this) {
        if (mView == null) {
            mView = view;
            
            // 1. 请求布局（触发遍历）
            requestLayout();
            
            // 2. 添加到 WMS
            res = mWindowSession.addToDisplayAsUser(
                mWindow,           // IWindow
                mWindowAttributes, // LayoutParams
                getHostVisibility(),
                mDisplay.getDisplayId(),
                userId,
                ...
                mSurfaceControl    // 输出参数，接收 SurfaceControl
            );
            
            // 3. 设置输入通道
            if (mInputChannel != null) {
                mInputEventReceiver = new WindowInputEventReceiver(
                    mInputChannel, Looper.myLooper());
            }
        }
    }
}
```

**WMS.addWindow()**

```java
// WindowManagerService.java
public int addWindow(Session session, IWindow client, ...,
        SurfaceControl outSurfaceControl, ...) {
    
    // 1. 权限检查
    int res = mPolicy.checkAddPermission(type, ...);
    if (res != ADD_OKAY) return res;
    
    synchronized (mGlobalLock) {
        // 2. 获取 DisplayContent
        final DisplayContent displayContent = getDisplayContentOrCreate(displayId);
        
        // 3. 获取或创建 WindowToken
        WindowToken token = displayContent.getWindowToken(attrs.token);
        if (token == null) {
            token = new WindowToken(this, attrs.token, type, ...);
        }
        
        // 4. 创建 WindowState
        final WindowState win = new WindowState(this, session, client, token, ...);
        
        // 5. 添加到窗口列表
        win.mToken.addWindow(win);
        
        // 6. 创建 SurfaceControl
        win.createSurfaceControl(outSurfaceControl);
        
        // 7. 更新焦点
        updateFocusedWindowLocked(UPDATE_FOCUS_NORMAL, false);
    }
    
    return ADD_OKAY;
}
```

### 3.2 performTraversals 详解

```java
// ViewRootImpl.java
private void performTraversals() {
    final View host = mView;
    
    // ============ 阶段 1: 预处理 ============
    if (mFirst) {
        // 首次遍历的特殊处理
    }
    
    // ============ 阶段 2: relayoutWindow ============
    if (mFirst || windowShouldResize || ...) {
        // 请求 WMS 布局
        relayoutResult = relayoutWindow(params, viewVisibility, ...);
        
        // 获取 Surface
        if (mSurfaceControl.isValid()) {
            mSurface.copyFrom(mSurfaceControl);
        }
    }
    
    // ============ 阶段 3: 测量 ============
    if (measureAgain) {
        performMeasure(childWidthMeasureSpec, childHeightMeasureSpec);
    }
    
    // ============ 阶段 4: 布局 ============
    if (didLayout) {
        performLayout(lp, mWidth, mHeight);
    }
    
    // ============ 阶段 5: 绘制 ============
    if (!cancelDraw) {
        performDraw();
    }
    
    mFirst = false;
}
```

### 4.2 invalidate vs requestLayout

```java
// View.java

// invalidate: 只标记需要重绘
public void invalidate() {
    invalidateInternal(0, 0, mRight - mLeft, mBottom - mTop, true, true);
}

void invalidateInternal(...) {
    // 设置 dirty 标记
    mPrivateFlags |= PFLAG_DIRTY;
    // 向上传递 dirty 区域
    final ViewParent p = mParent;
    if (p != null) {
        p.invalidateChild(this, damage);
    }
}

// requestLayout: 标记需要重新测量和布局
public void requestLayout() {
    // 设置 layout 标记
    mPrivateFlags |= PFLAG_FORCE_LAYOUT;
    mPrivateFlags |= PFLAG_INVALIDATED;
    // 向上传递请求
    if (mParent != null && !mParent.isLayoutRequested()) {
        mParent.requestLayout();
    }
}
```

### 5.2 关键代码

```java
// ViewRootImpl.java
void doDie() {
    synchronized (this) {
        if (mRemoved) return;
        mRemoved = true;
        
        if (mAdded) {
            // 1. 通知 View 树分离
            dispatchDetachedFromWindow();
        }
        
        // 2. 释放 Surface
        if (mSurface != null) {
            mSurface.release();
        }
        
        mAdded = false;
    }
    
    // 3. 从 WindowManagerGlobal 移除
    WindowManagerGlobal.getInstance().doRemoveView(this);
}

void dispatchDetachedFromWindow() {
    // 1. 分离输入通道
    if (mInputEventReceiver != null) {
        mInputEventReceiver.dispose();
        mInputEventReceiver = null;
    }
    
    // 2. 从 WMS 移除
    try {
        mWindowSession.remove(mWindow);
    } catch (RemoteException e) {
    }
    
    // 3. 通知 View
    mView.dispatchDetachedFromWindow();
}
```

```java
// WindowManagerService.java
void removeWindow(Session session, IWindow client) {
    synchronized (mGlobalLock) {
        WindowState win = windowForClientLocked(session, client, false);
        if (win != null) {
            win.removeIfPossible();
        }
    }
}

// WindowState.java
void removeIfPossible() {
    // 如果有动画，等动画结束
    if (mAnimatingExit) {
        return;
    }
    
    removeImmediately();
}

void removeImmediately() {
    // 1. 从 Token 移除
    mToken.removeChild(this);
    
    // 2. 销毁 Surface
    destroySurface();
    
    // 3. 移除 InputChannel
    mInputChannel.dispose();
    
    // 4. 更新焦点
    mWmService.updateFocusedWindowLocked(...);
}
```

boolean isVisibleLw() {
    return mHasSurface           // 有 Surface
        && mPolicyVisibility     // 策略允许显示

## 7. 生命周期回调

### 7.1 应用层回调

| 阶段 | 回调方法 | 说明 |
|-----|---------|------|
| 创建 | `View.onAttachedToWindow()` | View 附加到窗口 |
| 显示 | `View.onWindowVisibilityChanged()` | 窗口可见性变化 |
| 焦点 | `View.onWindowFocusChanged()` | 窗口焦点变化 |
| 销毁 | `View.onDetachedFromWindow()` | View 从窗口分离 |

### 7.2 回调顺序

```
窗口创建和显示：
Activity.onCreate()
  └─► setContentView()
Activity.onStart()
Activity.onResume()
  └─► addView()
      └─► View.onAttachedToWindow()
          └─► View.onWindowVisibilityChanged(VISIBLE)
              └─► View.onWindowFocusChanged(true)

窗口销毁：
Activity.onPause()
  └─► View.onWindowFocusChanged(false)
Activity.onStop()
  └─► View.onWindowVisibilityChanged(GONE)
Activity.onDestroy()
  └─► removeView()
      └─► View.onDetachedFromWindow()
```

## 8. 本章小结

本文介绍了窗口的完整生命周期：

1. **创建阶段**：addView → addWindow → 创建 WindowState/Layer
2. **显示阶段**：performTraversals → relayoutWindow → 首帧显示
3. **更新阶段**：invalidate/requestLayout → 帧更新
4. **销毁阶段**：removeView → removeWindow → 销毁 Layer
