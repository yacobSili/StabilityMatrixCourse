# Android 窗口系统系列文章（共 20 篇）

## 系列概述

本系列从架构师视角全面介绍 Android 窗口系统，涵盖应用层、Framework 层、Native 层和硬件层，详细阐述窗口从创建到显示的完整流程、模块交互关系、以及常见问题的分析与调试方法。

### 设计原则

1. **全局架构视角**：第一篇建立 App → Framework → SurfaceFlinger → HWC 的完整视图
2. **模块关系清晰**：明确上下游、平级交互、模块职责边界
3. **中等深度**：聚焦架构设计、关键机制、模块交互，不过度深入实现细节
4. **问题导向**：专门章节介绍黑屏、卡顿、丢帧等常见问题的分析方法
