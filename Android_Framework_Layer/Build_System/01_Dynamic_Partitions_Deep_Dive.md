## 1. 动态分区的设计原理

动态分区（Dynamic Partitions）是Android 10+引入的新分区系统，使用super分区作为容器，内部包含多个逻辑分区。

### 1.1 设计目标

1. **灵活性**：支持动态调整分区大小
2. **空间利用**：多个逻辑分区共享super分区空间
3. **OTA便利**：可以添加或删除逻辑分区，无需修改分区表

### 1.2 核心概念

- **Super分区**：物理分区，作为容器
- **逻辑分区**：super分区内的虚拟分区
- **分区组**：逻辑分区的分组（用于A/B槽位）

---

## 2. super分区的内部结构

super分区使用LVM（Logical Volume Manager）风格的结构：

```
Super分区布局：
┌─────────────────────────────────────┐
│  Metadata (64KB)                    │  ← 分区元数据
├─────────────────────────────────────┤
│  Logical Partition 1 (system_a)     │
├─────────────────────────────────────┤
│  Logical Partition 2 (system_b)     │
├─────────────────────────────────────┤
│  Logical Partition 3 (vendor_a)     │
├─────────────────────────────────────┤
│  ...                                │
└─────────────────────────────────────┘
```

## 3. 逻辑分区管理

### 3.1 分区组

逻辑分区组织成分区组，用于A/B槽位：

- `group_a`：Slot A的分区组
- `group_b`：Slot B的分区组

### 3.2 分区创建

逻辑分区在编译时创建，在OTA更新时可以动态调整大小。

---

## 4. 分区大小动态调整

### 4.1 调整机制

OTA更新时可以：
- 增加分区大小
- 减少分区大小
- 添加新分区
- 删除旧分区

### 4.2 调整限制

- 总大小不能超过super分区
- 每个组的大小有限制

## 5. lpmake/lpdump工具使用

### 5.1 lpmake - 创建super分区

```bash
lpmake \
    --device-size 17179869184 \
    --metadata-size 65536 \
    --metadata-slots 2 \
    --partition system_a:readonly:3221225472:group_a \
    --partition system_b:readonly:3221225472:group_b \
    --group group_a:8589934592 \
    --group group_b:8589934592 \
    --sparse \
    --output super.img
```

### 5.2 lpdump - 查看super分区

```bash
# 查看super分区内容
lpdump super.img

# 在设备上查看（需要root）
adb shell lpdump /dev/block/by-name/super
```

## 6. 动态分区与静态分区对比

| 特性 | 静态分区 | 动态分区 |
|------|---------|---------|
| **分区大小** | 固定 | 可调整 |
| **分区表** | GPT分区表 | super分区内 |
| **OTA更新** | 需要重新分区 | 可以动态调整 |
| **空间利用** | 可能浪费 | 高效利用 |

---

## 总结

动态分区系统要点：

1. **Super分区**：作为逻辑分区的容器
2. **逻辑分区**：可以动态调整大小
3. **分区组**：用于A/B槽位管理
4. **灵活性**：支持OTA时动态调整
