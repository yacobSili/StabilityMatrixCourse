## 1. 分区大小计算原则

1. **基于实际大小**：编译产物大小 × 1.2-1.3
2. **对齐要求**：必须是擦除块大小的倍数
3. **A/B槽位**：每个槽位都需要空间

---

## 2. 各分区大小估算

| 分区 | 估算方法 | 典型大小 |
|------|---------|---------|
| system | 编译产物 × 1.3 | 2-4GB |
| vendor | HAL大小 × 1.2 | 200MB-1GB |
| boot | 内核+ramdisk | 32-128MB |

---

## 3. 文件系统开销

ext4文件系统开销：
- 日志：约5%
- inode表：约1-2%
- 预留空间：约5%

---

## 4. 动态分区大小规划

对于super分区：
```
super_size = sum(所有逻辑分区大小) + 预留空间(10-20%)
```

---

## 总结

分区大小计算要点：

1. **预留空间**：为更新预留20-30%
2. **对齐要求**：必须是块大小的倍数
3. **A/B槽位**：需要双倍空间
