# 分区与挂载点映射关系

## 学习目标

- 理解分区表到块设备的映射
- 掌握分区挂载时机
- 理解 fstab 文件格式
- 了解分区挂载顺序
- 理解只读分区 vs 可写分区
- 掌握符号链接的使用

## 背景介绍

Android 设备上的分区需要挂载到文件系统的特定位置才能被访问。理解分区与挂载点的映射关系，是理解 Android 文件系统结构、启动流程和系统行为的基础。从 System-as-root 开始，挂载机制发生了重要变化。

## 核心概念

### 分区到块设备的映射

#### 分区表

**GPT 分区表**：
- 每个分区有唯一的 GUID
- 分区有名称（如 system, vendor）
- 分区映射到块设备（如 /dev/block/sda1）

#### 块设备路径

**传统路径**：
```
/dev/block/sda1
/dev/block/sda2
...
```

**符号链接**（推荐）：
```
/dev/block/by-name/system -> /dev/block/sda1
/dev/block/by-name/vendor -> /dev/block/sda2
...
```

**查看分区映射**：
```bash
# 查看所有分区
lsblk

# 查看符号链接
ls -la /dev/block/by-name/

# 查看分区信息
cat /proc/partitions
```

### fstab 文件

#### 文件位置

**多个位置**：
- `device/<vendor>/<device>/fstab.*` - 设备配置
- `/vendor/etc/fstab.*` - Vendor 分区
- `/system/etc/fstab.*` - System 分区（已废弃）

#### fstab 格式

**基本格式**：
```text
<src>                                    <mnt_point>  <type>  <mnt_flags>  <fs_mgr_flags>
```

**字段说明**：
1. **src**：源设备（分区）
2. **mnt_point**：挂载点
3. **type**：文件系统类型
4. **mnt_flags**：挂载标志
5. **fs_mgr_flags**：文件系统管理器标志

#### 示例

**fstab.qcom**：
```text
#<src>                                    <mnt_point>  <type>  <mnt_flags>  <fs_mgr_flags>
/dev/block/by-name/system                 /system      ext4    ro,barrier=1 wait,avb=vbmeta_system,logical,first_stage_mount
/dev/block/by-name/vendor                /vendor      ext4    ro,barrier=1 wait,avb=vbmeta_vendor,logical,first_stage_mount
/dev/block/by-name/product               /product     ext4    ro,barrier=1 wait,avb=vbmeta_product,logical,first_stage_mount
/dev/block/by-name/system_ext            /system_ext  ext4    ro,barrier=1 wait,avb=vbmeta_system_ext,logical,first_stage_mount
/dev/block/by-name/userdata              /data        ext4    noatime,nosuid,nodev,barrier=1,noauto_da_alloc wait,check,formattable,resize,quota,fileencryption=aes-256-xts:aes-256-xts,keydirectory=/metadata/vold/metadata_encryption,quota,reservedsize=128M,logical
/dev/block/by-name/cache                 /cache       ext4    noatime,nosuid,nodev,barrier=1 wait,check,formattable
```

#### fs_mgr_flags 详解

**常用标志**：
- `wait` - 等待设备就绪
- `check` - 检查文件系统
- `formattable` - 可以格式化
- `first_stage_mount` - 第一阶段挂载
- `avb=vbmeta_*` - AVB 验证
- `logical` - 逻辑分区（动态分区）
- `fileencryption=*` - 文件加密
- `quota` - 配额支持
- `resize` - 可以调整大小

### 分区挂载时机

#### 启动阶段

**第一阶段挂载**（First Stage Mount）：
- 在 Init 进程早期执行
- 挂载必要的分区（system, vendor 等）
- 使用 `first_stage_mount` 标志

**第二阶段挂载**（Second Stage Mount）：
- 在 Init 进程后期执行
- 挂载其他分区（data, cache 等）
- 使用 `wait` 标志

#### 挂载顺序

**典型顺序**：
1. **System** - 系统框架（System-as-root 下作为根）
2. **Vendor** - 厂商实现
3. **Product** - 产品内容
4. **System_ext** - 系统扩展
5. **Odm** - ODM 内容
6. **Data** - 用户数据
7. **Cache** - 缓存

### 只读分区 vs 可写分区

#### 只读分区

**特点**：
- 挂载为只读（`ro` 标志）
- 内容在构建时确定
- 运行时不能修改

**典型分区**：
- `system` - 系统框架
- `vendor` - 厂商实现
- `product` - 产品内容
- `system_ext` - 系统扩展
- `odm` - ODM 内容

**原因**：
1. **完整性保护**：防止系统文件被篡改
2. **性能优化**：只读文件系统可以优化
3. **安全**：配合 AVB 验证

#### 可写分区

**特点**：
- 挂载为读写（`rw` 标志）
- 内容可以动态修改
- 需要文件系统检查

**典型分区**：
- `data` - 用户数据
- `cache` - 缓存

**文件系统特性**：
- 支持配额
- 支持加密
- 支持调整大小

### System-as-root 的特殊处理

#### 传统方式

**Ramdisk 作为根**：
```
Kernel → Ramdisk (/) → 挂载 system/ → 切换根到 system/
```

**挂载点**：
- `/` - Ramdisk
- `/system` - System 分区

#### System-as-root

**System 分区作为根**：
```
Kernel → System (/) → 直接启动
```

**挂载点**：
- `/` - System 分区
- `/system` - 符号链接到 `/`（兼容性）

**实现方式**：
- System 分区直接挂载为根
- 创建符号链接 `/system -> /` 保持兼容

### 符号链接的使用

#### 兼容性符号链接

**System-as-root 下**：
```bash
/system -> /
/vendor -> /vendor
/product -> /product
```

**目的**：
- 保持应用和脚本的兼容性
- 允许代码使用 `/system` 路径

#### 查看符号链接

```bash
# 查看根目录符号链接
ls -la / | grep "^l"

# 查看具体链接
readlink -f /system
readlink -f /vendor
```

### 动态分区的挂载

#### 逻辑分区

**超级分区**：
```
super/
├── system_a/
├── system_b/
├── vendor_a/
├── vendor_b/
└── ...
```

**挂载**：
- 使用 `logical` 标志
- 通过逻辑分区名称挂载
- 支持 A/B 切换

#### 挂载示例

```text
/dev/block/by-name/system_a  /system  ext4  ro  wait,logical,first_stage_mount
```

### 挂载信息查看

#### 查看当前挂载

```bash
# 查看所有挂载
mount

# 查看特定分区
mount | grep system
mount | grep vendor

# 查看挂载表
cat /proc/mounts
```

#### 查看分区信息

```bash
# 查看分区大小
df -h

# 查看块设备
lsblk

# 查看分区详情
fdisk -l /dev/block/sda
```

## 实际应用

### 分析 fstab 文件

```bash
# 查看设备 fstab
cat device/<vendor>/<device>/fstab.*

# 查看运行时的 fstab
cat /vendor/etc/fstab.*
```

### 验证挂载

```bash
# 检查分区是否挂载
mount | grep -E "system|vendor|data"

# 检查挂载标志
mount | grep system
# 应该看到 ro（只读）标志
```

### 调试挂载问题

```bash
# 查看启动日志
dmesg | grep -i mount

# 查看 Init 日志
logcat -b all | grep -i mount

# 手动挂载测试
mount -t ext4 /dev/block/by-name/system /mnt
```

## 总结

### 核心要点

1. **分区通过符号链接映射**到块设备
2. **fstab 定义**分区挂载信息
3. **挂载分两个阶段**：第一阶段和第二阶段
4. **只读分区**保护系统完整性
5. **System-as-root** 下 System 分区作为根

### 关键概念

- **fstab**：分区挂载配置
- **first_stage_mount**：早期挂载
- **logical**：逻辑分区（动态分区）
- **avb**：验证启动
