# 进程调度基础架构

## 学习目标

- 理解 Linux 调度器的整体架构
- 掌握调度实体（sched_entity）和调度类（sched_class）的概念
- 理解运行队列（runqueue）的结构和管理
- 了解 schedule() 函数的工作流程
- 理解调度时机和触发条件

## 概述

Linux 调度器负责决定哪个进程获得 CPU 时间。调度器采用模块化设计，支持多种调度策略：

| 调度类 | 调度策略 | 适用场景 |
|-------|---------|---------|
| Stop | - | 最高优先级，用于停止 CPU |
| Deadline | SCHED_DEADLINE | 硬实时任务 |
| Realtime | SCHED_FIFO, SCHED_RR | 软实时任务 |
| CFS | SCHED_NORMAL, SCHED_BATCH, SCHED_IDLE | 普通进程 |
| Idle | - | 空闲任务 |

## 五、schedule() - 调度入口

### schedule() 函数

```c
// kernel/sched/core.c
asmlinkage __visible void __sched schedule(void)
{
    struct task_struct *tsk = current;
    
    // 调度前的准备工作
    sched_submit_work(tsk);
    
    do {
        // 禁用抢占
        preempt_disable();
        
        // 执行调度
        __schedule(SM_NONE);
        
        // 启用抢占（不立即检查重新调度）
        sched_preempt_enable_no_resched();
    } while (need_resched());
    
    // 调度后的清理工作
    sched_update_worker(tsk);
}
```

### __schedule() 核心函数

```c
// kernel/sched/core.c
static void __sched notrace __schedule(unsigned int sched_mode)
{
    struct task_struct *prev, *next;
    unsigned long *switch_count;
    struct rq_flags rf;
    struct rq *rq;
    int cpu;
    
    // 1. 获取当前 CPU 和运行队列
    cpu = smp_processor_id();
    rq = cpu_rq(cpu);
    prev = rq->curr;
    
    // 2. 调度统计
    switch_count = &prev->nivcsw;
    
    // 3. 锁定运行队列
    rq_lock(rq, &rf);
    
    // 4. 更新运行队列时钟
    update_rq_clock(rq);
    
    // 5. 如果当前进程需要出队
    if (!preempt_count() && prev->state) {
        if (signal_pending_state(prev->state, prev)) {
            // 有待处理信号，恢复为 RUNNING
            prev->state = TASK_RUNNING;
        } else {
            // 从运行队列移除
            deactivate_task(rq, prev, DEQUEUE_SLEEP | DEQUEUE_NOCLOCK);
        }
        switch_count = &prev->nvcsw;
    }
    
    // 6. 选择下一个任务
    next = pick_next_task(rq, prev, &rf);
    
    // 7. 清除重新调度标志
    clear_tsk_need_resched(prev);
    clear_preempt_need_resched();
    
    // 8. 上下文切换
    if (likely(prev != next)) {
        // 更新切换计数
        rq->nr_switches++;
        (*switch_count)++;
        
        // 设置新的当前任务
        RCU_INIT_POINTER(rq->curr, next);
        
        // 执行上下文切换
        rq = context_switch(rq, prev, next, &rf);
    } else {
        // 没有切换，解锁
        rq_unlock_irq(rq, &rf);
    }
    
    // 9. 负载均衡回调
    balance_callback(rq);
}
```

### pick_next_task() - 选择下一个任务

```c
// kernel/sched/core.c
static inline struct task_struct *pick_next_task(struct rq *rq,
                                                  struct task_struct *prev,
                                                  struct rq_flags *rf)
{
    const struct sched_class *class;
    struct task_struct *p;
    
    // 优化：大多数情况下是 CFS 任务
    if (likely(prev->sched_class <= &fair_sched_class &&
               rq->nr_running == rq->cfs.h_nr_running)) {
        // 只有 CFS 任务
        p = pick_next_task_fair(rq, prev, rf);
        if (unlikely(p == RETRY_TASK))
            goto restart;
        
        // 如果没有 CFS 任务，使用 idle
        if (!p) {
            put_prev_task(rq, prev);
            p = pick_next_task_idle(rq);
        }
        
        return p;
    }

restart:
    // 遍历所有调度类
    for_each_class(class) {
        p = class->pick_next_task(rq);
        if (p)
            return p;
    }
    
    // 不应该到达这里
    BUG();
}
```

## 六、调度时机

### 主动调度

进程主动放弃 CPU：

```c
// 1. 显式调用 schedule()
schedule();

// 2. 等待事件
wait_event_interruptible(wq, condition);

// 3. 睡眠
msleep(100);
schedule_timeout(timeout);

// 4. 让出 CPU
yield();
sched_yield();

// 5. 等待 IO
wait_for_completion(&done);
```

### 被动调度（抢占）

设置 TIF_NEED_RESCHED 标志后，在以下时机检查并调度：

```c
// 1. 时钟中断（时间片用完）
void scheduler_tick(void)
{
    // ...
    curr->sched_class->task_tick(rq, curr, 0);
    // task_tick 可能设置 need_resched
}

// 2. 唤醒进程（更高优先级）
void wake_up_process(struct task_struct *p)
{
    // ...
    check_preempt_curr(rq, p, 0);
    // 可能设置 need_resched
}

// 3. 优先级变化
void set_user_nice(struct task_struct *p, long nice)
{
    // ...
    // 可能设置 need_resched
}
```

### 抢占检查点

```c
// 1. 系统调用返回
// arch/arm64/kernel/entry.S
ret_to_user:
    // 检查 TIF_NEED_RESCHED
    
// 2. 中断返回
el1_irq:
    // 检查 TIF_NEED_RESCHED

if (need_resched())
```

### sysctl 参数

```bash
# CFS 调度器参数
/proc/sys/kernel/sched_latency_ns              # 调度延迟（默认 24ms）
/proc/sys/kernel/sched_min_granularity_ns      # 最小时间片（默认 3ms）
/proc/sys/kernel/sched_wakeup_granularity_ns   # 唤醒粒度（默认 4ms）
/proc/sys/kernel/sched_migration_cost_ns       # 迁移代价（默认 500us）
/proc/sys/kernel/sched_nr_migrate              # 每次迁移的最大任务数

# RT 调度器参数
/proc/sys/kernel/sched_rt_period_us            # RT 周期（默认 1s）
/proc/sys/kernel/sched_rt_runtime_us           # RT 运行时间（默认 950ms）
```

### 参数说明

| 参数 | 说明 | 默认值 |
|-----|------|-------|
| sched_latency_ns | 所有就绪进程运行一遍的目标时间 | 24ms |
| sched_min_granularity_ns | 单个进程的最小运行时间 | 3ms |
| sched_wakeup_granularity_ns | 唤醒进程抢占的最小 vruntime 差值 | 4ms |
| sched_migration_cost_ns | 迁移任务的缓存代价估计 | 500us |

## 总结

### 核心要点

1. **调度类**：
   - Stop > Deadline > RT > CFS > Idle
   - 每个调度类实现 sched_class 接口

2. **调度实体**：
   - sched_entity（CFS）
   - sched_rt_entity（RT）
   - sched_dl_entity（Deadline）

3. **运行队列**：
   - 每 CPU 一个 struct rq
   - 包含 CFS、RT、DL 子队列

4. **调度流程**：
   - schedule() → __schedule() → pick_next_task() → context_switch()

5. **调度时机**：
   - 主动：调用 schedule()
   - 被动：need_resched 标志
