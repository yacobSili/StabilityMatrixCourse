# 第1篇：开篇——Device Mapper是什么？为什么它是Linux/安卓存储的“隐形骨架”？

你是否曾对安卓手机的**动态分区**感到好奇？为什么`system`分区可以在不重新格式化的情况下变大变小？
你是否在排查安卓启动问题时，见过`dm-verity verification failed`的报错？它是如何守护系统完整性的？
你是否知道Linux系统的**LVM逻辑卷**、**LUKS全盘加密**，其底层依赖的是同一个核心框架？

这个隐藏在Linux和安卓存储栈深处，支撑起无数高级特性的“幕后英雄”，就是我们今天要揭开面纱的主角——**Device Mapper（简称DM）**。

作为一名安卓系统开发工程师，如果你正在处理存储相关的问题（如分区布局、性能调优、加密安全），那么DM层是你绕不开的核心关卡。本系列文章将带你从概念到源码，从原理到实战，彻底征服DM层。

## 一、DM层的本质：不是“功能”，而是“创造功能的框架”

首先，我们需要打破一个常见的误区：**Device Mapper不是一个具体的存储功能（如“加密”或“快照”），而是一个通用的块设备虚拟化框架。**

它的核心思想可以用**“乐高积木”**来类比：
- **Linux内核Block层**提供了最基础的“积木块”（物理磁盘、Bio/Request机制）。
- **DM层**提供了一套“积木拼接规则”和“多功能连接件”。
- **具体的存储功能**（加密、快照、线性映射）则是通过不同的“连接件”（DM Target）组合基础积木块实现的。

**官方定义**：
Device Mapper是Linux内核2.6版本后引入的一种**内核态块设备映射框架**。它允许用户空间通过简单的映射表，将一个或多个底层块设备（物理磁盘、分区、甚至其他DM设备）映射为一个全新的、抽象的**逻辑块设备**（DM设备）。

对上层应用（如文件系统、应用程序）来说，这个DM设备看起来和一个普通的物理磁盘毫无区别，可以被挂载、格式化、读写。但在底层，DM层会拦截所有发往该设备的IO请求，并根据映射表的规则进行处理（如地址转换、数据加密、重定向等）。

## 二、为什么需要DM层？核心价值在于“解耦”与“灵活”

在DM层出现之前，如果你想实现磁盘加密，可能需要修改文件系统代码；如果你想实现逻辑卷管理，可能需要修改内核Block层。这种紧耦合的方式导致代码难以维护，功能难以扩展。

DM层的出现解决了这些问题，其核心价值体现在以下三个方面：

### 1. 解耦“存储功能”与“物理设备/文件系统”
DM层将存储功能的实现（如加密、快照）从物理设备驱动和文件系统中剥离出来，形成独立的模块（Target驱动）。
- **文件系统**只需要关心如何管理文件，无需关心数据是否加密。
- **物理设备驱动**只需要关心如何与硬件通信，无需关心数据是否需要快照。
- **DM层**作为中间层，负责将这些功能“插入”到IO流程中。

### 2. 实现“功能的动态组合与复用”
通过组合不同的DM Target，你可以为一个逻辑设备叠加多个功能。例如：
- **“加密 + 线性映射”**：实现一个加密的逻辑分区。
- **“快照 + 加密 + 线性映射”**：实现一个可快照的加密逻辑分区。
  这种组合方式极其灵活，无需为每一种组合编写全新的代码。

### 3. 提供统一的用户空间管理接口
DM层通过`/dev/mapper/control`字符设备和`ioctl`命令，为用户空间提供了一套统一的管理接口。用户可以使用`dmsetup`等工具，在不重启系统、不卸载文件系统的情况下，动态地创建、修改、删除DM设备和它们的映射规则。

## 三、DM层的应用全景：Linux与安卓的“必选项”

DM层的应用范围非常广泛，从桌面Linux到服务器，再到安卓移动设备，它无处不在。

### 1. Linux系统中的经典应用
| 应用场景 | 核心DM Target | 功能描述 |
| :--- | :--- | :--- |
| **LVM (逻辑卷管理)** | `linear`, `striped`, `mirror` | 将多个物理磁盘合并为一个卷组，动态创建/调整逻辑卷大小，实现磁盘空间的灵活管理。 |
| **LUKS (Linux统一密钥设置)** | `crypt` | 实现磁盘级别的加密，保护数据隐私，是Linux系统全盘加密的标准方案。 |
| **多路径IO (MPIO)** | `multipath` | 为存储设备提供多条访问路径，实现冗余备份和负载均衡，提升存储可靠性和性能。 |
| **存储快照** | `snapshot`, `thin` | 为逻辑卷创建只读或可写快照，用于数据备份、测试环境快速恢复等场景。 |
| **精简配置 (Thin Provisioning)** | `thin` | 按需分配物理存储空间，提升存储利用率，常用于虚拟化环境。 |

### 2. 安卓系统中的核心应用（重点！）
对于安卓系统开发工程师来说，DM层更是不可或缺的“基础设施”，它支撑了安卓的多个核心特性：

| 安卓核心特性 | 核心DM Target | 功能描述 |
| :--- | :--- | :--- |
| **动态分区 (Dynamic Partitions)** | `linear` | Android 10+ 引入的分区方案。将`super`一个大物理分区，通过DM线性映射，动态划分为`system`、`vendor`、`product`等多个逻辑分区。解决了传统静态分区空间分配不灵活、OTA升级空间不足的问题。 |
| **系统完整性校验 (dm-verity)** | `verity` | Android 4.4+ 引入。在启动时验证`/system`等关键分区的哈希值，确保系统文件未被篡改。是实现安卓安全启动（Secure Boot）和Google SafetyNet认证的关键。 |
| **全盘加密 (FDE) / 文件级加密 (FBE)** | `crypt` | 基于`dm-crypt`实现。对`/data`分区进行加密，保护用户隐私数据。FBE更是实现了对单个应用文件的独立加密。 |
| **虚拟A/B分区 (Virtual A/B)** | `snapshot` | Android 11+ 引入。在OTA升级时，利用DM快照技术，在不影响当前系统运行的情况下，将更新写入快照分区。升级失败可直接回滚，实现“无缝升级”。 |

可以说，**没有DM层，就没有现代安卓系统的灵活性、安全性和可维护性。**

## 四、DM层在存储栈中的准确位置

要理解DM层的工作原理，首先要明确它在Linux存储栈中的位置。

### 1. 存储栈架构图
```mermaid
graph TD
    A[应用层] --> B[VFS层]
    B --> C[文件系统层 (ext4, f2fs, erofs)]
    C --> D[Block层 (通用块层 + IO调度器)]
    D --> E[DM层 (Device Mapper) - 可堆叠块设备]
    E --> F[物理块设备驱动 (eMMC, UFS, NVMe)]
    F --> G[物理存储设备]

    style E fill:#e1f5fe,stroke:#01579b,stroke-width:2px
```

### 2. 关键定位解读
从架构图中可以清晰地看到：
1.  **DM层位于Block层之上，物理设备驱动之上**：它是一个**“可堆叠”**的块设备层。
2.  **DM层是Block层的“消费者”也是“生产者”**：
    -   **作为消费者**：它接收来自上层文件系统、经过Block层调度的IO请求（Bio）。
    -   **作为生产者**：它处理完IO请求后，将其转发给底层的物理块设备（同样经过Block层）。
3.  **DM层对上层透明**：文件系统和应用程序看到的DM设备，就是一个普通的块设备。
4.  **DM层可嵌套**：一个DM设备可以作为另一个DM设备的底层设备。例如，一个加密的DM设备（dm-crypt）可以基于一个快照DM设备（dm-snapshot）创建。

**常见误区纠正**：DM层**不是**在VFS层和Block层之间，而是完全处于Block层的范畴内，是对Block层功能的一种扩展和虚拟化。

## 五、DM层与LVM的关系：框架与应用

很多初学者容易混淆DM和LVM，这里需要明确区分：
-   **DM是“底层框架”**：它提供了实现块设备映射和IO处理的基础能力。
-   **LVM是“上层应用”**：它是基于DM框架实现的一套逻辑卷管理工具集。

**形象的比喻**：
-   **DM层**就像是**“操作系统的内核”**，提供了进程管理、内存管理等基础功能。
-   **LVM**就像是**“办公软件”**，它基于操作系统内核提供的功能，为用户提供了文档编辑、表格处理等具体应用。

LVM的所有功能（如创建PV、VG、LV），最终都是通过向DM层发送指令，创建相应的DM设备和映射表来实现的。除了LVM，cryptsetup（LUKS工具）、multipath-tools（多路径工具）等也都是DM层的上层应用。

## 六、快速上手：如何验证系统中的DM层？

说了这么多，让我们动手验证一下DM层是否存在于你的系统中。

### 1. Linux系统 (桌面/服务器)
```bash
# 1. 检查DM内核模块是否加载
lsmod | grep dm_mod

# 2. 查看系统中已创建的DM设备
dmsetup ls

# 3. 查看某个DM设备的详细映射表（例如 /dev/mapper/vg0-lv0）
dmsetup table vg0-lv0

# 4. 查看DM设备的状态信息
dmsetup info -c
```

### 2. 安卓设备 (需要ADB连接)
```bash
# 1. 进入安卓shell
adb shell

# 2. 查看DM设备（通常在 /dev/block/dm-*）
ls -l /dev/block/dm-*

# 3. 查看DM设备的映射表
dmsetup table

# 4. 查看DM设备的详细信息
dmsetup info -c

# 5. 查看动态分区信息（基于DM linear）
ls -l /dev/block/by-name/
```

如果你在安卓设备上看到了`/dev/block/dm-0`、`/dev/block/dm-1`等设备，并且`dmsetup table`输出了包含`linear`、`verity`或`crypt`的映射规则，那就说明DM层正在你的设备中默默工作。

## 七、总结与预告

### 核心要点回顾
1.  **Device Mapper是一个通用的块设备虚拟化框架**，而非具体功能。
2.  它的核心价值在于**解耦存储功能与底层设备/文件系统**，并提供**灵活的功能组合**。
3.  DM层是**现代安卓系统的基石**，支撑了动态分区、dm-verity、加密、虚拟A/B分区等核心特性。
4.  它位于**Linux存储栈的Block层之上**，是一个可堆叠的块设备层。
5.  LVM、LUKS等是基于DM层实现的**上层应用**。
