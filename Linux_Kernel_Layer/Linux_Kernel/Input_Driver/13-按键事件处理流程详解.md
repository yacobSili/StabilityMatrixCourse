# 按键事件处理流程详解

## 引言

按键事件 (KeyEvent) 是 Android Input 系统中的重要事件类型，包括物理按键（电源键、音量键）和虚拟按键（遥控器、键盘）。本文将详细分析按键事件从 InputDispatcher 到应用层的完整处理流程。

---

## 1. 按键事件分发概览

### 1.1 分发流程

```
InputDispatcher
    │
    │ dispatchKeyLocked()
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                          策略拦截                                        │
│  interceptKeyBeforeQueueing()   - 入队前拦截 (电源键等)                 │
│  interceptKeyBeforeDispatching() - 分发前拦截 (快捷键等)                │
└─────────────────────────────────────────────────────────────────────────┘
    │
    │ InputChannel
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           应用进程                                       │
│  ViewRootImpl.WindowInputEventReceiver                                  │
│      │                                                                  │
│      ▼                                                                  │
│  InputStage 链                                                          │
│  ├── NativePreImeInputStage                                            │
│  ├── ViewPreImeInputStage     → dispatchKeyEventPreIme()               │
│  ├── ImeInputStage            → IME 处理                               │
│  ├── EarlyPostImeInputStage                                            │
│  ├── NativePostImeInputStage                                           │
│  └── ViewPostImeInputStage    → dispatchKeyEvent()                     │
│          │                                                              │
│          ▼                                                              │

│          │                                                              │
│          ▼                                                              │
│  View Hierarchy                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 系统按键处理

| 按键 | 处理位置 | 处理方式 |
|-----|---------|---------|
| POWER | PhoneWindowManager | 休眠/唤醒 |
| VOLUME | PhoneWindowManager | 音量调节 |
| HOME | PhoneWindowManager | 返回桌面 |
| BACK | Activity | 默认 finish() |
| MENU | Activity | 打开选项菜单 |

## 2. InputDispatcher 按键分发

### 2.1 dispatchKeyLocked

```cpp
// frameworks/native/services/inputflinger/dispatcher/InputDispatcher.cpp

bool InputDispatcher::dispatchKeyLocked(nsecs_t currentTime,
                                         std::shared_ptr<KeyEntry> entry,
                                         DropReason* dropReason,
                                         nsecs_t* nextWakeupTime) {
    // 1. 检查是否需要重复
    if (entry->repeatCount == 0 && entry->action == AKEY_EVENT_ACTION_DOWN) {
        // 首次按下, 可能需要重复
    }
    
    // 2. 策略拦截 - 入队后、分发前
    if (entry->interceptKeyResult == KeyEntry::InterceptKeyResult::UNKNOWN) {
        if (entry->policyFlags & POLICY_FLAG_PASS_TO_USER) {
            // 需要策略判断
            postCommandLocked(
                &InputDispatcher::doInterceptKeyBeforeDispatchingLockedInterruptible);
            return false;  // 等待策略处理
        } else {
            entry->interceptKeyResult = KeyEntry::InterceptKeyResult::CONTINUE;
        }
    }
    
    // 3. 检查策略拦截结果
    if (entry->interceptKeyResult == KeyEntry::InterceptKeyResult::SKIP) {
        // 被策略拦截, 不分发
        *dropReason = DropReason::POLICY;
        return true;
    }
    
    // 4. 查找焦点窗口
    std::vector<InputTarget> inputTargets;
    InputEventInjectionResult injectionResult = 
        findFocusedWindowTargetsLocked(currentTime, *entry, 
                                       inputTargets, nextWakeupTime);
    
    if (injectionResult == InputEventInjectionResult::PENDING) {
        return false;  // 等待焦点窗口
    }
    
    if (injectionResult != InputEventInjectionResult::SUCCEEDED) {
        *dropReason = DropReason::NO_FOCUS;
        return true;
    }
    
    // 5. 分发到目标窗口
    dispatchEventLocked(currentTime, entry, inputTargets);
    return true;
}
```

### 2.2 按键重复

```cpp
// 处理按键重复
void InputDispatcher::doNotifyKeyLocked(KeyEntry* entry) {
    if (entry->action == AKEY_EVENT_ACTION_DOWN && entry->repeatCount == 0) {
        // 设置重复定时器
        if (mKeyRepeatState.lastKeyEntry != nullptr) {
            // 取消之前的重复
            releaseKeyEntry(mKeyRepeatState.lastKeyEntry);
        }
        
        mKeyRepeatState.lastKeyEntry = entry;
        mKeyRepeatState.lastKeyEntry->refCount++;
        
        // 设置首次重复延迟
        mKeyRepeatState.nextRepeatTime = 
            entry->eventTime + mConfig.keyRepeatDelay;
    } else if (entry->action == AKEY_EVENT_ACTION_UP) {
        // 按键释放, 取消重复
        if (mKeyRepeatState.lastKeyEntry != nullptr &&
            mKeyRepeatState.lastKeyEntry->keyCode == entry->keyCode) {
            releaseKeyEntry(mKeyRepeatState.lastKeyEntry);
            mKeyRepeatState.lastKeyEntry = nullptr;
        }
    }
}

// 生成重复事件
void InputDispatcher::synthesizeKeyRepeatLocked(nsecs_t currentTime) {
    KeyEntry* entry = mKeyRepeatState.lastKeyEntry;
    
    // 创建重复事件
    std::unique_ptr<KeyEntry> repeatEntry = 
        std::make_unique<KeyEntry>(mIdGenerator.nextId(),
                                   currentTime,
                                   entry->deviceId,
                                   entry->source,
                                   entry->displayId,
                                   entry->policyFlags,
                                   AKEY_EVENT_ACTION_DOWN,
                                   entry->flags,
                                   entry->keyCode,
                                   entry->scanCode,
                                   entry->metaState,
                                   entry->repeatCount + 1,
                                   entry->downTime);
    
    // 加入队列
    enqueueInboundEventLocked(std::move(repeatEntry));
    
    // 设置下次重复时间
    mKeyRepeatState.nextRepeatTime = currentTime + mConfig.keyRepeatTimeout;
}
```

## 3. 策略拦截

### 3.1 interceptKeyBeforeQueueing

```java
// frameworks/base/services/core/java/com/android/server/policy/PhoneWindowManager.java

@Override
public int interceptKeyBeforeQueueing(KeyEvent event, int policyFlags) {
    final boolean down = event.getAction() == KeyEvent.ACTION_DOWN;
    final int keyCode = event.getKeyCode();
    
    int result = 0;
    boolean isWakeKey = (policyFlags & WindowManagerPolicy.FLAG_WAKE) != 0;
    
    switch (keyCode) {
        case KeyEvent.KEYCODE_POWER: {
            // 电源键
            result &= ~ACTION_PASS_TO_USER;
            isWakeKey = false;
            
            if (down) {
                interceptPowerKeyDown(event, mInteractive);
            } else {
                interceptPowerKeyUp(event, mInteractive);
            }
            break;
        }
        
        case KeyEvent.KEYCODE_VOLUME_DOWN:
        case KeyEvent.KEYCODE_VOLUME_UP:
        case KeyEvent.KEYCODE_VOLUME_MUTE: {
            // 音量键
            if (down) {
                // 检查是否需要截屏 (音量减 + 电源键)
                if (mScreenshotChordEnabled && keyCode == KeyEvent.KEYCODE_VOLUME_DOWN
                        && mPowerKeyDown) {
                    mScreenshotChordVolumeDownKeyTriggered = true;
                }
                
                // 音量调节
                mAudioManager.adjustStreamVolume(...);
            }
            
            if (mUseTvRouting || mHandleVolumeKeysInWM) {
                result |= ACTION_PASS_TO_USER;
            }
            break;
        }
        
        case KeyEvent.KEYCODE_HEADSETHOOK:
        case KeyEvent.KEYCODE_MEDIA_PLAY:
        case KeyEvent.KEYCODE_MEDIA_PAUSE:
        case KeyEvent.KEYCODE_MEDIA_PLAY_PAUSE: {
            // 媒体按键
            if (MediaSessionLegacyHelper.getHelper(mContext)
                    .isGlobalPriorityActive()) {
                result &= ~ACTION_PASS_TO_USER;
            }
            break;
        }
        
        default:
            // 其他按键传递给应用
            result |= ACTION_PASS_TO_USER;
    }
    
    // 处理唤醒
    if (isWakeKey && !mInteractive) {
        result |= ACTION_WAKE_UP;
    }
    
    return result;
}
```

### 3.2 interceptKeyBeforeDispatching

```java
@Override
public long interceptKeyBeforeDispatching(IBinder focusedToken,
                                          KeyEvent event, int policyFlags) {
    final int keyCode = event.getKeyCode();
    final int action = event.getAction();
    final int repeatCount = event.getRepeatCount();
    final int metaState = event.getMetaState();
    
    // 已处理的按键不分发
    if ((policyFlags & WindowManagerPolicy.FLAG_PASS_TO_USER) == 0) {
        return -1;
    }
    
    // 快捷键处理
    if (action == KeyEvent.ACTION_DOWN && repeatCount == 0) {
        // 截屏快捷键
        if (keyCode == KeyEvent.KEYCODE_S && 
            (metaState & KeyEvent.META_META_ON) != 0) {
            takeScreenshot();
            return -1;
        }
        
        // 搜索快捷键
        if (keyCode == KeyEvent.KEYCODE_SEARCH) {
            launchAssistAction();
            return -1;
        }
    }
    
    // HOME 键
    if (keyCode == KeyEvent.KEYCODE_HOME) {
        if (action == KeyEvent.ACTION_UP) {
            if (!mHomeConsumed) {
                launchHomeFromHotKey();
            }
        } else if (action == KeyEvent.ACTION_DOWN) {
            mHomeConsumed = false;
        }
        return -1;
    }
    
    // BACK 键长按
    if (keyCode == KeyEvent.KEYCODE_BACK) {
        if (action == KeyEvent.ACTION_DOWN && repeatCount == 0) {
            mBackKeyHandled = false;
        } else if (action == KeyEvent.ACTION_UP) {
            if (!mBackKeyHandled) {
                // 正常 BACK 处理, 传递给应用
                return 0;
            }
        }
    }
    
    return 0;  // 正常分发
}
```

## 4. 应用层事件处理

### 4.1 InputStage 链

```java
// frameworks/base/core/java/android/view/ViewRootImpl.java

void setView(View view, WindowManager.LayoutParams attrs, ...) {
    // 构建 InputStage 链
    mSyntheticInputStage = new SyntheticInputStage();
    InputStage viewPostImeStage = new ViewPostImeInputStage(mSyntheticInputStage);
    InputStage nativePostImeStage = new NativePostImeInputStage(viewPostImeStage);
    InputStage earlyPostImeStage = new EarlyPostImeInputStage(nativePostImeStage);
    InputStage imeStage = new ImeInputStage(earlyPostImeStage);
    InputStage viewPreImeStage = new ViewPreImeInputStage(imeStage);
    InputStage nativePreImeStage = new NativePreImeInputStage(viewPreImeStage);
    
    mFirstInputStage = nativePreImeStage;
    mFirstPostImeInputStage = earlyPostImeStage;
}

// 事件分发入口
private void deliverInputEvent(QueuedInputEvent q) {
    InputStage stage;
    
    if (q.shouldSendToSynthesizer()) {
        stage = mSyntheticInputStage;
    } else {
        stage = mFirstInputStage;
    }
    
    if (stage != null) {
        stage.deliver(q);
    } else {
        finishInputEvent(q);
    }
}
```

### 4.2 ViewPostImeInputStage

```java
final class ViewPostImeInputStage extends InputStage {
    @Override
    protected int onProcess(QueuedInputEvent q) {
        if (q.mEvent instanceof KeyEvent) {
            return processKeyEvent(q);
        }
        return FORWARD;
    }
    
    private int processKeyEvent(QueuedInputEvent q) {
        final KeyEvent event = (KeyEvent) q.mEvent;
        
        // 分发给 View 层次结构
        if (mView.dispatchKeyEvent(event)) {
            return FINISH_HANDLED;
        }
        
        // View 未处理, 尝试默认处理
        if (shouldDropInputEvent(q)) {
            return FINISH_NOT_HANDLED;
        }
        
        // 音量键后备处理
        if (event.getKeyCode() == KeyEvent.KEYCODE_VOLUME_UP ||
            event.getKeyCode() == KeyEvent.KEYCODE_VOLUME_DOWN) {
            int direction = (event.getKeyCode() == KeyEvent.KEYCODE_VOLUME_UP) 
                ? AudioManager.ADJUST_RAISE : AudioManager.ADJUST_LOWER;
            
            if (event.getAction() == KeyEvent.ACTION_DOWN) {
                mAudioManager.adjustSuggestedStreamVolume(direction, ...);
            }
            return FINISH_HANDLED;
        }
        
        return FORWARD;
    }
}
```

### 4.3 DecorView 分发

```java
// frameworks/base/core/java/com/android/internal/policy/DecorView.java

@Override
public boolean dispatchKeyEvent(KeyEvent event) {
    final int keyCode = event.getKeyCode();
    final int action = event.getAction();
    
    // 允许 Window 拦截
    if (mWindow.superDispatchKeyEvent(event)) {
        return true;
    }
    
    // 如果 Activity 存在, 分发给 Activity
    final Callback cb = mWindow.getCallback();
    if (cb != null && !mWindow.isDestroyed()) {
        return cb.dispatchKeyEvent(event);
    }
    
    return false;
}
```

### 4.4 Activity 处理

```java
// frameworks/base/core/java/android/app/Activity.java

public boolean dispatchKeyEvent(KeyEvent event) {
    // 窗口优先处理
    if (getWindow().superDispatchKeyEvent(event)) {
        return true;
    }
    
    // Activity 自身处理
    return onKeyDown(event.getKeyCode(), event) ||
           onKeyUp(event.getKeyCode(), event);
}

public boolean onKeyDown(int keyCode, KeyEvent event) {
    switch (keyCode) {
        case KeyEvent.KEYCODE_BACK:
            // BACK 键
            if (event.getRepeatCount() == 0) {
                if (mDefaultBackCallback != null) {
                    // 使用回调
                    mDefaultBackCallback.handleOnBackPressed();
                    return true;
                }
            }
            break;
            
        case KeyEvent.KEYCODE_MENU:
            // MENU 键
            if (event.getRepeatCount() == 0 && mActionBar != null) {
                mActionBar.openOptionsMenu();
                return true;
            }
            break;
    }
    
    return false;
}

public boolean onKeyUp(int keyCode, KeyEvent event) {
    switch (keyCode) {
        case KeyEvent.KEYCODE_BACK:
            // BACK 键释放
            onBackPressed();
            return true;
    }
    
    return false;
}

public void onBackPressed() {
    // 检查 Fragment 回退栈
    FragmentManager fragmentManager = mFragments.getSupportFragmentManager();
    if (fragmentManager.popBackStackImmediate()) {
        return;
    }
    
    // 结束 Activity
    finishAfterTransition();
}
```

## 5. View 层按键处理

### 5.1 View.dispatchKeyEvent

```java
// frameworks/base/core/java/android/view/View.java

public boolean dispatchKeyEvent(KeyEvent event) {
    // 焦点监听器
    if (mOnKeyListener != null && (mViewFlags & ENABLED_MASK) == ENABLED
            && mOnKeyListener.onKey(this, event.getKeyCode(), event)) {
        return true;
    }
    
    // 事件处理
    if (event.dispatch(this, mAttachInfo != null
            ? mAttachInfo.mKeyDispatchState : null, this)) {
        return true;
    }
    
    return false;
}

// KeyEvent.dispatch 调用
public boolean onKeyDown(int keyCode, KeyEvent event) {
    // 处理确认键
    if (keyCode == KeyEvent.KEYCODE_ENTER || keyCode == KeyEvent.KEYCODE_DPAD_CENTER) {
        if ((mViewFlags & ENABLED_MASK) == ENABLED && isClickable()) {
            setPressed(true);
            return true;
        }
    }
    
    return false;
}

public boolean onKeyUp(int keyCode, KeyEvent event) {
    if (keyCode == KeyEvent.KEYCODE_ENTER || keyCode == KeyEvent.KEYCODE_DPAD_CENTER) {
        if ((mViewFlags & ENABLED_MASK) == ENABLED && isPressed()) {
            setPressed(false);
            
            if (!mHasPerformedLongPress) {
                performClick();
            }
            return true;
        }
    }
    
    return false;
}
```

// frameworks/base/core/java/android/view/ViewGroup.java

@Override
public boolean dispatchKeyEvent(KeyEvent event) {
    // 先分发给焦点子 View
    if ((mPrivateFlags & (PFLAG_FOCUSED | PFLAG_HAS_BOUNDS))
            == (PFLAG_FOCUSED | PFLAG_HAS_BOUNDS)) {
        if (super.dispatchKeyEvent(event)) {
            return true;
        }
    } else if (mFocused != null && (mFocused.mPrivateFlags & PFLAG_HAS_BOUNDS)
            == PFLAG_HAS_BOUNDS) {
        if (mFocused.dispatchKeyEvent(event)) {
            return true;
        }
    }
    
    return false;
}

## 6. 方向键与焦点导航

### 6.1 焦点查找

```java
// 处理方向键
public boolean onKeyDown(int keyCode, KeyEvent event) {
    // 方向键焦点导航
    int direction = 0;
    switch (keyCode) {
        case KeyEvent.KEYCODE_DPAD_LEFT:
            direction = FOCUS_LEFT;
            break;
        case KeyEvent.KEYCODE_DPAD_RIGHT:
            direction = FOCUS_RIGHT;
            break;
        case KeyEvent.KEYCODE_DPAD_UP:
            direction = FOCUS_UP;
            break;
        case KeyEvent.KEYCODE_DPAD_DOWN:
            direction = FOCUS_DOWN;
            break;
        case KeyEvent.KEYCODE_TAB:
            direction = event.isShiftPressed() ? FOCUS_BACKWARD : FOCUS_FORWARD;
            break;
    }
    
    if (direction != 0) {
        View focused = findFocus();
        if (focused != null) {
            View next = focused.focusSearch(direction);
            if (next != null && next != focused) {
                next.requestFocus(direction);
                return true;
            }
        }
    }
    
    return false;
}
```

## 7. 本章小结

按键事件处理流程：

| 阶段 | 处理内容 |
|-----|---------|
| InputDispatcher | 查找焦点窗口, 策略拦截 |
| PhoneWindowManager | 系统按键拦截 (电源/音量/HOME) |
| InputStage 链 | IME 处理, View 分发 |
| DecorView | 窗口级分发 |
| Activity | BACK/MENU 处理 |
| View | 焦点 View 处理, 方向键导航 |

关键点：
1. 系统按键在 PhoneWindowManager 中拦截处理
2. InputStage 链实现了灵活的事件处理流水线
3. Activity 提供 BACK/MENU 默认行为
4. 方向键用于焦点导航
