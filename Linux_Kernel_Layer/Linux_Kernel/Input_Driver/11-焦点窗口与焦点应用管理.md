# 焦点窗口与焦点应用管理

## 引言

焦点管理是 Android Input 系统的核心机制之一，决定了输入事件（特别是按键事件）应该发送到哪个窗口。本文将深入分析窗口焦点和应用焦点的管理机制，以及焦点切换的流程。

---

## 1. 焦点概念

### 1.1 焦点类型

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          焦点类型                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  焦点应用 (Focused Application)                                         │
│  ├── 定义: 当前与用户交互的应用                                         │
│  ├── 用途: ANR 检测的目标, 决定输入延迟容忍度                          │
│  └── 管理者: ActivityManagerService + WindowManagerService              │
│                                                                         │
│  焦点窗口 (Focused Window)                                              │
│  ├── 定义: 接收按键事件的窗口                                           │
│  ├── 用途: 按键事件的分发目标                                           │
│  └── 管理者: WindowManagerService                                       │
│                                                                         │
│  触摸窗口 (Touched Window)                                              │
│  ├── 定义: 接收触摸事件的窗口                                           │
│  ├── 用途: 触摸事件的分发目标                                           │
│  └── 确定方式: 坐标命中测试                                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 焦点与事件分发的关系

```
按键事件 (KeyEvent)
    │
    ▼
┌────────────────────────────────┐
│  InputDispatcher               │
│  findFocusedWindowTargetLocked │
│        │                       │
│        ▼                       │
│  焦点窗口 (mFocusedWindow)     │
└────────────────────────────────┘
    │
    ▼
  目标窗口


触摸事件 (MotionEvent)
    │
    ▼
┌────────────────────────────────┐
│  InputDispatcher               │
│  findTouchedWindowTargetLocked │
│        │                       │
│        ▼                       │
│  坐标命中测试                  │
│  (遍历窗口栈, 检查触摸区域)   │
└────────────────────────────────┘
    │
    ▼
  目标窗口(s)
```

## 3. 焦点设置流程

### 3.1 WMS 设置焦点

```java
// frameworks/base/services/core/java/com/android/server/wm/WindowManagerService.java

void updateFocusedWindowLocked(int mode, boolean updateInputWindows) {
    // 找到新的焦点窗口
    WindowState newFocus = computeFocusedWindowLocked();
    
    if (mCurrentFocus != newFocus) {
        // 焦点变化
        final WindowState oldFocus = mCurrentFocus;
        mCurrentFocus = newFocus;
        
        // 通知焦点变化
        if (oldFocus != null) {
            oldFocus.reportFocusChangedSerialized(false);
        }
        if (newFocus != null) {
            newFocus.reportFocusChangedSerialized(true);
        }
        
        // 更新 InputDispatcher
        if (updateInputWindows) {
            mInputManagerCallback.updateInputWindowsLw(false);
        }
        
        // 更新焦点应用
        updateFocusedApplicationLocked();
    }
}

private WindowState computeFocusedWindowLocked() {
    // 遍历所有显示器
    for (int i = mRoot.getChildCount() - 1; i >= 0; i--) {
        final DisplayContent dc = mRoot.getChildAt(i);
        
        // 在每个显示器上查找焦点窗口
        final WindowState win = dc.findFocusedWindow();
        if (win != null) {
            return win;
        }
    }
    return null;
}
```

### 3.2 DisplayContent 查找焦点窗口

```java
// frameworks/base/services/core/java/com/android/server/wm/DisplayContent.java

WindowState findFocusedWindow() {
    // 按 Z-Order 从上到下遍历窗口
    return getWindow(w -> {
        // 检查窗口是否可聚焦
        if (!w.canReceiveKeys()) {
            return false;
        }
        
        // 检查窗口是否可见
        if (!w.isVisible()) {
            return false;
        }
        
        // 检查是否是顶部全屏窗口
        if (w.inFullscreenTaskContainer()) {
            return true;
        }
        
        return true;
    });
}
```

### 3.3 WindowState.canReceiveKeys

```java
// frameworks/base/services/core/java/com/android/server/wm/WindowState.java

boolean canReceiveKeys() {
    // 检查标志
    return isVisibleOrAdding()
            && (mViewVisibility == View.VISIBLE)
            && ((mAttrs.flags & WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE) == 0)
            && (mActivityRecord == null || mActivityRecord.windowsAreFocusable());
}
```

## 4. 通知 InputDispatcher

### 4.1 setInputWindows

```java
// frameworks/base/services/core/java/com/android/server/wm/InputMonitor.java

void updateInputWindowsLw(boolean force) {
    // 收集所有窗口信息
    final InputWindowHandle[] windowHandles = collectInputWindowHandles();
    
    // 发送到 InputDispatcher
    mService.mInputManager.setInputWindows(windowHandles, mDisplayId);
    
    // 设置焦点窗口
    if (mFocusedWindow != null) {
        mService.mInputManager.setFocusedWindow(mFocusedWindow.mInputWindowHandle);
    }
}

private InputWindowHandle[] collectInputWindowHandles() {
    List<InputWindowHandle> handles = new ArrayList<>();
    
    // 遍历窗口
    forAllWindows(w -> {
        if (w.mInputChannel != null) {
            InputWindowHandle handle = w.mInputWindowHandle;
            
            // 更新窗口信息
            handle.name = w.getName();
            handle.displayId = mDisplayId;
            handle.token = w.mInputChannel.getToken();
            
            // 更新区域
            handle.frame.set(w.getFrame());
            handle.touchableRegion.set(w.getTouchableRegion());
            
            // 更新标志
            handle.visible = w.isVisible();
            handle.focusable = w.canReceiveKeys();
            
            handles.add(handle);
        }
    }, true /* traverseTopToBottom */);
    
    return handles.toArray(new InputWindowHandle[0]);
}
```

### 4.2 Native 层处理

```cpp
// frameworks/native/services/inputflinger/dispatcher/InputDispatcher.cpp

void InputDispatcher::setInputWindows(
        const std::unordered_map<int32_t, std::vector<sp<WindowInfoHandle>>>& 
            handlesPerDisplay) {
    
    { // acquire lock
        std::scoped_lock _l(mLock);
        
        for (const auto& [displayId, handles] : handlesPerDisplay) {
            setInputWindowsLocked(handles, displayId);
        }
    } // release lock
    
    // 唤醒分发线程
    mLooper->wake();
}

void InputDispatcher::setInputWindowsLocked(
        const std::vector<sp<WindowInfoHandle>>& windowInfoHandles,
        int32_t displayId) {
    
    // 更新窗口句柄列表
    mWindowHandlesByDisplay[displayId] = windowInfoHandles;
    
    // 查找焦点窗口的新句柄
    sp<WindowInfoHandle>& focusedWindowHandle = 
        mFocusedWindowHandlesByDisplay[displayId];
    
    if (focusedWindowHandle != nullptr) {
        // 检查焦点窗口是否还在列表中
        bool found = false;
        for (const sp<WindowInfoHandle>& handle : windowInfoHandles) {
            if (handle->getToken() == focusedWindowHandle->getToken()) {
                focusedWindowHandle = handle;
                found = true;
                break;
            }
        }
        
        if (!found) {
            // 焦点窗口已移除
            focusedWindowHandle = nullptr;
        }
    }
}
```

### 4.3 setFocusedWindow

```cpp
void InputDispatcher::setFocusedWindow(const FocusRequest& request) {
    { // acquire lock
        std::scoped_lock _l(mLock);
        
        sp<WindowInfoHandle> newFocusedWindow = 
            getWindowHandleLocked(request.token, request.displayId);
        
        if (newFocusedWindow == nullptr && request.token != nullptr) {
            // 窗口不在列表中, 等待
            mPendingFocusRequests[request.displayId] = request;
            return;
        }
        
        // 设置焦点窗口
        sp<WindowInfoHandle>& oldFocusedWindow = 
            mFocusedWindowHandlesByDisplay[request.displayId];
        
        if (oldFocusedWindow != newFocusedWindow) {
            // 发送焦点变化事件
            if (oldFocusedWindow != nullptr) {
                // 发送失去焦点事件
                sp<Connection> connection = 
                    getConnectionLocked(oldFocusedWindow->getToken());
                if (connection != nullptr) {
                    sendFocusChangedEntryLocked(connection, false);
                }
            }
            
            if (newFocusedWindow != nullptr) {
                // 发送获得焦点事件
                sp<Connection> connection = 
                    getConnectionLocked(newFocusedWindow->getToken());
                if (connection != nullptr) {
                    sendFocusChangedEntryLocked(connection, true);
                }
            }
            
            oldFocusedWindow = newFocusedWindow;
        }
    } // release lock
}
```

## 5. 焦点应用管理

### 5.1 更新焦点应用

```java
// frameworks/base/services/core/java/com/android/server/wm/ActivityTaskManagerService.java

void updateTopResumedActivityIfNeeded() {
    final ActivityRecord topResumed = getTopResumedActivity();
    
    if (mTopResumedActivity != topResumed) {
        // 焦点应用变化
        final ActivityRecord prev = mTopResumedActivity;
        mTopResumedActivity = topResumed;
        
        // 通知 InputManager
        updateFocusedApplication();
    }
}

private void updateFocusedApplication() {
    synchronized (mGlobalLock) {
        final DisplayContent dc = mRootWindowContainer.getDefaultDisplay();
        final ActivityRecord focusedApp = dc.mFocusedApp;
        
        if (focusedApp != null) {
            // 创建 InputApplicationHandle
            final InputApplicationHandle handle = focusedApp.getInputApplicationHandle();
            
            // 设置到 InputDispatcher
            mInputManager.setFocusedApplication(dc.mDisplayId, handle);
        }
    }
}
```

### 5.3 Native 层设置

```cpp
void InputDispatcher::setFocusedApplication(
        int32_t displayId,
        const std::shared_ptr<InputApplicationHandle>& inputApplicationHandle) {
    
    { // acquire lock
        std::scoped_lock _l(mLock);
        
        std::shared_ptr<InputApplicationHandle> oldApp = 
            getValueByKey(mFocusedApplicationHandlesByDisplay, displayId);
        
        if (oldApp == inputApplicationHandle) {
            return;  // 没有变化
        }
        
        // 更新焦点应用
        if (inputApplicationHandle != nullptr) {
            mFocusedApplicationHandlesByDisplay[displayId] = inputApplicationHandle;
        } else {
            mFocusedApplicationHandlesByDisplay.erase(displayId);
        }
        
        // 重置无焦点窗口超时
        resetNoFocusedWindowTimeoutLocked();
        
    } // release lock
    
    // 唤醒分发线程
    mLooper->wake();
}
```

## 6. 焦点切换场景

### 6.1 Activity 切换

```
场景: Activity A → Activity B

1. Activity B 启动
   │
   ▼
2. AMS 更新 Top Resumed Activity
   │
   ▼
3. WMS 更新窗口栈
   │
   ├──► setFocusedApplication(B)  → InputDispatcher
   │
   └──► Activity B 窗口就绪后
        │
        ▼
4. WMS 计算新焦点窗口
   │
   ▼
5. setFocusedWindow(B's Window) → InputDispatcher
   │
   ▼
6. InputDispatcher 发送 FocusEvent
   │
   ├──► Activity A's Window: hasFocus = false
   │
   └──► Activity B's Window: hasFocus = true
```

### 6.2 Dialog 显示

```
场景: Activity 显示 Dialog

1. Activity 创建 Dialog
   │
   ▼
2. Dialog.show() → 添加窗口到 WMS
   │
   ▼
3. WMS 更新窗口栈
   │
   ▼
4. 如果 Dialog 可聚焦:
   │
   ├──► setFocusedWindow(Dialog) → InputDispatcher
   │
   └──► InputDispatcher 发送 FocusEvent
        │
        ├──► Activity's Window: hasFocus = false
        │
        └──► Dialog's Window: hasFocus = true
   
5. 焦点应用不变 (同一应用)
```

### 6.3 输入法显示

```
场景: 点击 EditText 显示输入法

1. 点击 EditText
   │
   ▼
2. View 请求焦点
   │
   ▼
3. InputMethodManager 显示输入法
   │
   ▼
4. 输入法窗口添加到 WMS
   │
   ▼
5. WMS 更新窗口栈 (输入法窗口在最上层)
   │
   ▼
6. 焦点窗口处理:
   - 输入法窗口通常设置 FLAG_NOT_FOCUSABLE
   - 焦点仍保持在 EditText 所在窗口
   
7. 按键事件流向:
   EditText 窗口 → 输入法处理 (通过 InputConnection)
```

## 7. ANR 与焦点

### 7.1 无焦点窗口 ANR

```cpp
// 当有焦点应用但无焦点窗口时, 启动 ANR 计时
void InputDispatcher::resetNoFocusedWindowTimeoutLocked() {
    // 如果有焦点应用, 设置超时时间
    for (const auto& [displayId, appHandle] : mFocusedApplicationHandlesByDisplay) {
        if (appHandle != nullptr) {
            mNoFocusedWindowTimeoutTime = currentTime() + 
                appHandle->getDispatchingTimeout();
            break;
        }
    }
}

// 在分发循环中检查
bool InputDispatcher::dispatchKeyLocked(...) {
    // 查找焦点窗口
    int32_t injectionResult = findFocusedWindowTargetsLocked(...);
    
    if (injectionResult == INPUT_EVENT_INJECTION_PENDING) {
        // 没有焦点窗口, 检查是否超时
        if (currentTime() >= mNoFocusedWindowTimeoutTime) {
            // 触发 ANR
            std::shared_ptr<InputApplicationHandle> focusedApp = 
                getValueByKey(mFocusedApplicationHandlesByDisplay, displayId);
            
            onAnrLocked(focusedApp);
        }
        return false;  // 等待
    }
    
    // ...
}
```

```cpp
void InputDispatcher::startDispatchCycleLocked(...) {
    // 发送事件
    // ...
    
    // 设置响应超时检测
    nsecs_t timeout = connection->inputChannel->getDispatchingTimeout();

void InputDispatcher::processAnrsLocked() {
    // 检查所有等待队列
    for (auto& [token, connection] : mConnectionsByToken) {
        if (!connection->waitQueue.empty()) {
            DispatchEntry* entry = connection->waitQueue.front();
            nsecs_t waitDuration = currentTime() - entry->deliveryTime;
            
            if (waitDuration > connection->inputChannel->getDispatchingTimeout()) {
                // 窗口无响应
                onAnrLocked(connection);
            }
        }
    }
}
```

## 8. 本章小结

焦点管理是 Input 系统的核心机制：

| 焦点类型 | 用途 | 管理者 |
|---------|------|--------|
| 焦点应用 | ANR 检测目标 | AMS/ATMS |
| 焦点窗口 | 按键事件目标 | WMS |
| 触摸窗口 | 触摸事件目标 | 命中测试 |

焦点切换流程：
1. 系统状态变化 (Activity 切换, Dialog 显示等)
2. WMS 重新计算焦点窗口
3. 调用 IMS 接口更新 InputDispatcher
4. InputDispatcher 发送 FocusEvent 通知窗口

关键点：
- 焦点应用和焦点窗口可能不同步 (如窗口创建中)
- 无焦点窗口时会触发 ANR 检测
- 触摸事件不依赖焦点, 而是通过坐标命中测试
