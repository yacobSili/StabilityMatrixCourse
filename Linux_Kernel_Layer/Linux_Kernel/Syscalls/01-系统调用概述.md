# 系统调用概述

## 学习目标

- 理解什么是系统调用及其作用
- 掌握用户空间与内核空间的区别
- 了解系统调用的分类和数量
- 熟悉常用系统调用列表
- 理解系统调用的基本流程

## 背景介绍

系统调用（System Call）是用户空间程序与内核空间交互的唯一接口。理解系统调用是深入理解 Linux 内核的基础，也是分析性能问题、调试系统行为的关键。

## 核心概念

### 什么是系统调用

**系统调用（System Call）** 是用户空间程序请求内核执行特定操作的接口。它是用户空间和内核空间之间的桥梁。

#### 系统调用的特点

1. **特权操作**：系统调用允许用户程序执行需要内核特权的操作
2. **统一接口**：提供统一的接口访问内核功能
3. **安全隔离**：通过系统调用实现用户空间和内核空间的安全隔离
4. **性能开销**：系统调用涉及用户态到内核态的切换，有一定的性能开销

### 为什么需要系统调用

#### 用户空间 vs 内核空间

Linux 系统将内存空间分为两部分：

1. **用户空间（User Space）**
   - 应用程序运行的空间
   - 权限受限，不能直接访问硬件
   - 不能执行特权指令
   - 内存地址范围：0x00000000 00000000 - 0x0000ffff ffffffff（ARM64）

2. **内核空间（Kernel Space）**
   - 内核代码运行的空间
   - 拥有最高权限
   - 可以访问所有硬件资源
   - 内存地址范围：0xffff0000 00000000 - 0xffffffff ffffffff（ARM64）

#### 隔离的必要性

**安全隔离**：
- 防止用户程序直接访问硬件
- 防止用户程序破坏系统
- 提供统一的资源管理

**稳定性保证**：
- 内核代码经过充分测试
- 统一的错误处理
- 防止应用程序崩溃影响系统

### 系统调用的作用

系统调用提供了以下功能：

1. **文件操作**：打开、读取、写入、关闭文件
2. **进程管理**：创建、终止、等待进程
3. **内存管理**：分配、释放、映射内存
4. **网络通信**：创建套接字、发送接收数据
5. **设备控制**：控制硬件设备
6. **系统信息**：获取系统状态和信息

## 系统调用的分类

### 按功能分类

#### 1. 文件操作类

**作用**：文件和设备的操作

**常用系统调用**：
- `open()` - 打开文件/设备
- `read()` - 读取数据
- `write()` - 写入数据
- `close()` - 关闭文件
- `lseek()` - 文件定位
- `ioctl()` - 设备控制
- `fcntl()` - 文件控制
- `stat()` / `fstat()` / `lstat()` - 获取文件状态
- `mmap()` / `munmap()` - 内存映射

#### 2. 进程管理类

**作用**：进程的创建、终止和管理

**常用系统调用**：
- `fork()` / `clone()` - 创建进程
- `execve()` - 执行程序
- `exit()` / `_exit()` - 进程退出
- `wait()` / `waitpid()` - 等待子进程
- `getpid()` / `getppid()` - 获取进程 ID
- `setuid()` / `getuid()` - 用户 ID 管理
- `setgid()` / `getgid()` - 组 ID 管理

#### 3. 内存管理类

**作用**：内存的分配、释放和管理

**常用系统调用**：
- `mmap()` / `munmap()` - 内存映射/取消映射
- `brk()` / `sbrk()` - 堆内存管理
- `mprotect()` - 内存保护
- `madvise()` - 内存建议
- `mincore()` - 内存驻留检查
- `mremap()` - 内存重映射

#### 4. 网络通信类

**作用**：网络套接字操作

**常用系统调用**：
- `socket()` - 创建套接字
- `bind()` - 绑定地址
- `listen()` - 监听连接
- `accept()` - 接受连接
- `connect()` - 建立连接
- `send()` / `recv()` - 发送/接收数据
- `sendto()` / `recvfrom()` - 发送/接收数据（UDP）
- `setsockopt()` / `getsockopt()` - 套接字选项

#### 5. 信号处理类

**作用**：信号的发送和处理

**常用系统调用**：
- `signal()` / `sigaction()` - 信号处理
- `kill()` / `tgkill()` - 发送信号
- `sigprocmask()` - 信号掩码
- `sigsuspend()` - 等待信号
- `rt_sigaction()` - 实时信号处理

#### 6. 同步原语类

**作用**：进程/线程同步

**常用系统调用**：
- `futex()` - 快速用户空间互斥锁
- `semop()` / `semget()` - 信号量
- `shmget()` / `shmat()` - 共享内存
- `msgget()` / `msgsnd()` / `msgrcv()` - 消息队列

#### 7. 系统信息类

**作用**：获取系统状态和信息

**常用系统调用**：
- `gettimeofday()` / `clock_gettime()` - 获取时间
- `uname()` - 获取系统信息
- `sysinfo()` - 系统信息
- `getrusage()` - 资源使用情况

### 按调用方式分类

#### 1. 直接系统调用

**方式**：直接使用系统调用号调用

**示例**：
```c
// ARM64 架构
mov x8, #__NR_open  // 系统调用号
svc #0              // 触发系统调用
```

#### 2. 通过 C 库封装

**方式**：通过 glibc 等 C 库封装

**示例**：
```c
int fd = open("file.txt", O_RDONLY);
```

C 库内部会调用系统调用。

## Linux 系统调用数量统计

### 系统调用总数

**Linux 5.15 内核**：
- 约 **400+** 个系统调用
- 不同架构可能有少量差异
- 部分系统调用是架构特定的

### 系统调用号分配

**ARM64 架构**：
- 系统调用号从 0 开始
- 通用系统调用：0-400+
- 架构特定系统调用：可能有额外编号

**查看系统调用号**：
```powershell
# 查看 ARM64 系统调用号定义
cd ../common
cat arch/arm64/include/uapi/asm/unistd.h
```

### 常用系统调用统计

根据实际使用频率，最常用的系统调用包括：

1. **文件操作**：open, read, write, close（使用频率最高）
2. **进程管理**：fork, execve, exit, wait
3. **内存管理**：mmap, munmap, brk
4. **网络通信**：socket, connect, send, recv
5. **系统信息**：gettimeofday, getpid

## 系统调用的基本流程

### 完整流程

```
用户程序
    ↓
C 库函数（如 open）
    ↓
系统调用包装函数（如 __open）
    ↓
触发系统调用（svc 指令）
    ↓
内核入口（entry.S）
    ↓
系统调用处理（syscall.c）
    ↓
查找系统调用表（sys_call_table）
    ↓
执行系统调用函数（如 sys_open）
    ↓
返回结果
    ↓
返回到用户空间
```

### 详细步骤

#### 1. 用户空间调用

**示例**：
```c
int fd = open("file.txt", O_RDONLY);
```

**过程**：
- 用户程序调用 C 库函数 `open()`
- C 库函数准备参数
- 调用系统调用包装函数

#### 2. 触发系统调用

**ARM64 架构**：
```assembly
mov x8, #__NR_open    // 系统调用号放入 x8
svc #0                // 触发系统调用异常
```

**过程**：
- 将系统调用号放入 x8 寄存器
- 参数放入 x0-x7 寄存器
- 执行 `svc` 指令触发异常

#### 3. 内核入口处理

**位置**：`../common/arch/arm64/kernel/entry.S`

**过程**：
- 异常处理程序被触发
- 保存用户空间寄存器
- 切换到内核栈
- 调用系统调用处理函数

#### 4. 系统调用分发

**位置**：`../common/arch/arm64/kernel/syscall.c`

**过程**：
- 从寄存器读取系统调用号
- 检查系统调用号有效性
- 从 `sys_call_table` 查找处理函数
- 调用对应的系统调用函数

#### 5. 执行系统调用

**过程**：
- 执行具体的系统调用实现
- 进行权限检查
- 执行实际操作
- 返回结果

#### 6. 返回用户空间

**过程**：
- 将返回值放入 x0 寄存器
- 恢复用户空间寄存器
- 返回到用户空间
- 继续执行用户程序

## 实际应用

### 查看系统调用

```bash
# 使用 strace 追踪系统调用
strace ls

# 查看特定进程的系统调用
strace -p <PID>

# 统计系统调用
strace -c ls
```

### 系统调用性能

**开销分析**：
- 用户态到内核态切换：~100-200 纳秒
- 系统调用本身执行时间：取决于具体操作
- 总开销：通常 < 1 微秒（简单系统调用）

**优化方法**：
- 减少系统调用次数
- 使用批处理操作
- 利用 vDSO 机制（某些系统调用）

## 与你的性能问题的关联

系统调用可能与你的性能问题相关：

1. **频繁的系统调用**：GO 应用可能频繁调用系统调用
2. **系统调用开销**：大量系统调用累积的开销
3. **内存管理系统调用**：mmap/brk 等调用可能影响性能
4. **文件 I/O 系统调用**：read/write 的性能

**分析方法**：
```bash
# 追踪 GO 应用的系统调用
strace -c <go_application>

# 查看系统调用频率
strace -c -e trace=mmap,brk,read,write <go_application>
```

## 总结

### 核心要点

1. **系统调用是用户空间和内核空间的桥梁**：提供安全的接口访问内核功能
2. **Linux 有 400+ 个系统调用**：覆盖文件、进程、内存、网络等各个方面
3. **系统调用有性能开销**：需要用户态到内核态的切换
4. **系统调用按功能分类**：文件操作、进程管理、内存管理等

### 关键概念

- **用户空间**：应用程序运行的空间，权限受限
- **内核空间**：内核运行的空间，拥有最高权限
- **系统调用号**：标识不同系统调用的数字
- **系统调用表**：存储系统调用处理函数的数组
