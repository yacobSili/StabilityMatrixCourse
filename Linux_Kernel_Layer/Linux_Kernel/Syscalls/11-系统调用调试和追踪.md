# 系统调用调试和追踪

## 学习目标

- 掌握 strace 工具的使用
- 了解 ftrace 追踪系统调用
- 理解 perf 性能分析
- 掌握内核日志中的系统调用信息
- 学习系统调用错误的诊断方法

## 背景介绍

调试和追踪系统调用是分析系统行为、排查问题、优化性能的重要手段。掌握这些工具和方法，有助于快速定位问题和理解系统行为。

## 核心概念

### strace 工具

#### 基本使用

```bash
# 追踪系统调用
strace <command>

# 追踪特定系统调用
strace -e trace=open,read,write <command>

# 统计系统调用
strace -c <command>

# 查看系统调用时间
strace -T <command>
```

#### 输出格式

**系统调用格式**：
```
系统调用名(参数) = 返回值
```

**示例**：
```
open("file.txt", O_RDONLY) = 3
read(3, "hello", 5) = 5
close(3) = 0
```

### ftrace 追踪

#### 使用 ftrace

```bash
# 启用系统调用追踪
echo 1 > /sys/kernel/debug/tracing/events/syscalls/enable

# 查看追踪结果
cat /sys/kernel/debug/tracing/trace
```

### perf 性能分析

#### 使用 perf

```bash
# 记录系统调用
perf record -e syscalls:sys_enter_* <command>

# 查看报告
perf report
```

## 实际应用

### 调试系统调用问题

```bash
# 追踪系统调用错误
strace -e trace=open,read,write <command> 2>&1 | grep -i error

# 查看系统调用参数
strace -v <command>
```

## 总结

### 核心要点

1. **strace 是最常用的工具**：简单易用，功能强大
2. **ftrace 提供内核级追踪**：更详细的内部信息
3. **perf 用于性能分析**：分析系统调用性能

### 关键工具

- **strace**：用户空间系统调用追踪
- **ftrace**：内核级追踪
- **perf**：性能分析工具
