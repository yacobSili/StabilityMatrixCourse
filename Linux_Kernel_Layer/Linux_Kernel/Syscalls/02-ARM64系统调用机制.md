# ARM64 系统调用机制深入解析

## 学习目标

- 理解 ARM64 架构下的系统调用指令（svc）
- 掌握从用户空间到内核空间的切换过程
- 了解系统调用入口点的实现
- 理解寄存器使用约定和参数传递
- 掌握 pt_regs 结构体的作用
- 理解系统调用返回机制
- 了解与 GKI 架构的关联

## 背景介绍

ARM64 架构下的系统调用实现有其特定的机制和优化。深入理解这些机制，有助于分析系统调用性能、调试系统调用问题，以及理解内核与用户空间的交互。

## 核心概念

### ARM64 系统调用指令

#### SVC 指令

**SVC (Supervisor Call)** 是 ARM64 架构中触发系统调用的指令。

**指令格式**：
```assembly
svc #0    // 触发系统调用异常
```

**作用**：
- 触发异常，从用户态切换到内核态
- 异常类型：Synchronous Exception（同步异常）
- 异常级别：从 EL0（用户态）切换到 EL1（内核态）

#### 系统调用号传递

**寄存器约定**：
- **x8 寄存器**：存储系统调用号
- **x0-x7 寄存器**：存储系统调用参数（最多 8 个参数）
- **x0 寄存器**：存储返回值

**示例**：
```assembly
// 调用 open("file.txt", O_RDONLY)
mov x0, #文件路径地址    // 第一个参数
mov x1, #O_RDONLY       // 第二个参数
mov x8, #__NR_open      // 系统调用号
svc #0                  // 触发系统调用
// x0 寄存器包含返回值（文件描述符）
```

### 从用户空间到内核空间的切换

#### 异常向量表

**位置**：`../common/arch/arm64/kernel/entry.S`

ARM64 使用异常向量表处理各种异常，包括系统调用。

**向量表结构**：
```assembly
// 用户态同步异常（包括系统调用）
el0_sync:
    kernel_entry 0
    mov x0, sp
    bl  el0_svc_handler
    b   ret_to_user
```

#### 异常处理流程

**步骤**：

1. **触发异常**
   - 用户程序执行 `svc #0` 指令
   - CPU 检测到同步异常
   - 跳转到异常向量表

2. **保存上下文**
   - 保存用户空间寄存器
   - 切换到内核栈
   - 保存程序状态寄存器（PSTATE）

3. **调用处理函数**
   - 调用 `el0_svc_handler`
   - 传递寄存器上下文

4. **执行系统调用**
   - 从寄存器读取系统调用号
   - 查找系统调用表
   - 执行系统调用函数

5. **返回用户空间**
   - 恢复用户空间寄存器
   - 返回到用户程序

### 系统调用入口点

#### entry.S 中的实现

**位置**：`../common/arch/arm64/kernel/entry.S`

**关键代码**：
```assembly
// 用户态同步异常处理（系统调用入口）
el0_sync:
    kernel_entry 0                    // 保存用户空间上下文
    mov x0, sp                        // 传递 pt_regs 指针
    bl  el0_svc_handler               // 调用系统调用处理函数
    b   ret_to_user                   // 返回到用户空间
```

**kernel_entry 宏**：
- 保存所有通用寄存器（x0-x30）
- 保存程序状态寄存器
- 切换到内核栈
- 设置异常上下文

#### el0_svc_handler 函数

**位置**：`../common/arch/arm64/kernel/syscall.c`

**函数签名**：
```c
void el0_svc_handler(struct pt_regs *regs)
```

**功能**：
- 从 pt_regs 读取系统调用号（regs->regs[8]）
- 调用 `el0_svc_common` 处理系统调用
- 处理系统调用追踪
- 处理信号

### 寄存器使用约定

#### 参数传递

**ARM64 ABI 约定**：

1. **系统调用号**：x8 寄存器
2. **参数 1-8**：x0-x7 寄存器
3. **返回值**：x0 寄存器
4. **错误码**：负数表示错误，绝对值是错误码

**示例**：
```c
// C 代码
int fd = open("file.txt", O_RDONLY);

// 对应的汇编（简化）
mov x0, #文件路径地址    // 第一个参数：路径
mov x1, #O_RDONLY       // 第二个参数：标志
mov x8, #__NR_open      // 系统调用号
svc #0                  // 触发系统调用
// x0 包含返回值（文件描述符或错误码）
```

#### 寄存器保存

**pt_regs 结构体**保存所有寄存器：

```c
struct pt_regs {
    union {
        struct user_pt_regs user_regs;
        struct {
            u64 regs[31];
            u64 sp;
            u64 pc;
            u64 pstate;
        };
    };
    u64 orig_x0;
    u64 syscallno;
    // ...
};
```

**保存的寄存器**：
- x0-x30：通用寄存器
- sp：栈指针
- pc：程序计数器
- pstate：程序状态寄存器

### pt_regs 结构体详解

#### 结构体定义

**位置**：`../common/arch/arm64/include/asm/ptrace.h`

**关键字段**：

1. **regs[31]**：通用寄存器数组
   - regs[0-7]：参数寄存器（x0-x7）
   - regs[8]：系统调用号（x8）
   - regs[9-30]：其他通用寄存器（x9-x30）

2. **sp**：栈指针

3. **pc**：程序计数器（返回地址）

4. **pstate**：程序状态寄存器

5. **orig_x0**：原始的第一个参数（用于重启系统调用）

6. **syscallno**：系统调用号

#### 使用场景

**系统调用处理**：
```c
// 从 pt_regs 读取系统调用号
int scno = regs->regs[8];

// 读取参数
unsigned long arg1 = regs->regs[0];
unsigned long arg2 = regs->regs[1];

// 设置返回值
regs->regs[0] = ret_value;
```

### 系统调用返回机制

#### ret_to_user

**位置**：`../common/arch/arm64/kernel/entry.S`

**功能**：
- 检查是否有待处理的信号
- 检查是否需要调度
- 恢复用户空间寄存器
- 返回到用户空间

**关键代码**：
```assembly
ret_to_user:
    disable_daif
    gic_prio_irq_setup tmp=x3
    user_exit_irqoff
    enable_step_tsk x0, x1
    kernel_exit 0
```

#### kernel_exit 宏

**功能**：
- 恢复用户空间寄存器
- 恢复程序状态寄存器
- 返回到用户空间（ERET 指令）

**ERET 指令**：
- 从 EL1（内核态）返回到 EL0（用户态）
- 恢复 PC 和 PSTATE
- 继续执行用户程序

### el0_svc_common 函数详解

#### 函数实现

**位置**：`../common/arch/arm64/kernel/syscall.c`

**关键代码**：
```c
static void el0_svc_common(struct pt_regs *regs, int scno, int sc_nr,
                           const syscall_fn_t syscall_table[])
{
    unsigned long flags = current_thread_info()->flags;
    
    regs->orig_x0 = regs->regs[0];
    regs->syscallno = scno;
    
    // 检查是否需要追踪
    if (has_syscall_work(flags)) {
        // 系统调用追踪
        if (syscall_trace_enter(regs)) {
            // 追踪后可能跳过系统调用
            return;
        }
    }
    
    // 调用系统调用
    invoke_syscall(regs, scno, sc_nr, syscall_table);
    
    // 系统调用返回后的处理
    if (has_syscall_work(flags)) {
        syscall_trace_exit(regs);
    }
}
```

#### invoke_syscall 函数

**功能**：
- 检查系统调用号有效性
- 从系统调用表查找处理函数
- 调用系统调用处理函数
- 设置返回值

**关键代码**：
```c
static void invoke_syscall(struct pt_regs *regs, unsigned int scno,
                           unsigned int sc_nr,
                           const syscall_fn_t syscall_table[])
{
    long ret;
    
    // 添加随机栈偏移（安全特性）
    add_random_kstack_offset();
    
    if (scno < sc_nr) {
        syscall_fn_t syscall_fn;
        // 使用 array_index_nospec 防止 Spectre 攻击
        syscall_fn = syscall_table[array_index_nospec(scno, sc_nr)];
        ret = __invoke_syscall(regs, syscall_fn);
    } else {
        ret = do_ni_syscall(regs, scno);
    }
    
    // 设置返回值
    syscall_set_return_value(current, regs, 0, ret);
}
```

### 安全机制

#### Spectre 缓解

**array_index_nospec**：
- 防止 Spectre 变体 1 攻击
- 在数组访问前插入屏障指令
- 防止推测执行泄露数据

**随机栈偏移**：
- 每次系统调用添加随机栈偏移
- 防止栈溢出攻击
- 增加攻击难度

#### 系统调用追踪

**ptrace 支持**：
- 支持 strace 等工具
- 可以追踪系统调用
- 可以修改系统调用参数和返回值

### 与 GKI 架构的关联

#### GKI 中的系统调用

**特点**：
- Generic Kernel 包含所有系统调用实现
- 系统调用是内核核心功能
- 不通过模块提供系统调用

#### 系统调用与 KMI

**关系**：
- 系统调用是内核向用户空间提供的接口
- KMI 是内核向模块提供的接口
- 两者都是内核接口，但面向不同对象

#### 性能考虑

**系统调用开销**：
- 用户态到内核态切换
- 寄存器保存和恢复
- 系统调用表查找
- 实际系统调用执行

**优化方法**：
- vDSO 机制（某些系统调用）
- 快速路径优化
- 减少不必要的系统调用

## 实际应用

### 调试系统调用

```bash
# 使用 strace 追踪系统调用
strace -e trace=open,read,write ls

# 查看系统调用参数
strace -v ls

# 查看系统调用时间
strace -T ls
```

### 分析系统调用性能

```bash
# 统计系统调用次数和时间
strace -c ls

# 查看特定系统调用
strace -e trace=mmap,brk ls
```

## 与你的性能问题的关联

ARM64 系统调用机制可能与你的性能问题相关：

1. **系统调用开销**：频繁的系统调用可能累积开销
2. **寄存器保存/恢复**：每次系统调用都需要保存和恢复寄存器
3. **栈操作**：内核栈操作可能影响性能
4. **安全机制开销**：Spectre 缓解机制可能增加开销

**分析方法**：
```bash
# 统计 GO 应用的系统调用
strace -c <go_application>

# 查看系统调用频率
strace -c -e trace=mmap,brk,read,write <go_application>
```

## 总结

### 核心要点

1. **ARM64 使用 SVC 指令触发系统调用**：从 EL0 切换到 EL1
2. **寄存器约定**：x8 存储系统调用号，x0-x7 存储参数，x0 存储返回值
3. **pt_regs 保存完整上下文**：包括所有寄存器和程序状态
4. **系统调用有安全机制**：包括 Spectre 缓解和栈随机化

### 关键机制

- **异常向量表**：处理系统调用异常
- **寄存器保存**：完整保存用户空间上下文
- **系统调用分发**：通过系统调用表查找处理函数
- **安全返回**：安全地返回到用户空间
