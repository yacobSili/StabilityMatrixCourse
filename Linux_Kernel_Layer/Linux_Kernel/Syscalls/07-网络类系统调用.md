# 网络类系统调用

## 学习目标

- 理解网络类系统调用的作用和分类
- 掌握 socket 系统调用的实现
- 理解 connect、bind、listen、accept 等系统调用
- 了解 send/recv 等数据传输系统调用
- 掌握网络系统调用与网络协议栈的关系

## 背景介绍

网络通信是现代应用程序的重要功能。理解网络类系统调用的实现，有助于分析网络性能、理解网络协议栈，以及优化网络应用。

## 核心概念

### 网络类系统调用分类

#### 1. 套接字创建和管理

- **socket()** - 创建套接字
- **bind()** - 绑定地址
- **listen()** - 监听连接
- **accept()** - 接受连接
- **connect()** - 建立连接
- **shutdown()** - 关闭连接
- **close()** - 关闭套接字（复用文件操作）

#### 2. 数据传输

- **send()** / **recv()** - TCP 数据传输
- **sendto()** / **recvfrom()** - UDP 数据传输
- **sendmsg()** / **recvmsg()** - 高级数据传输
- **sendmmsg()** / **recvmmsg()** - 批量数据传输

#### 3. 套接字选项

- **setsockopt()** - 设置套接字选项
- **getsockopt()** - 获取套接字选项

#### 4. 地址和连接管理

- **getsockname()** - 获取本地地址
- **getpeername()** - 获取对端地址

### socket 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(socket, int, family, int, type, int, protocol)
```

**参数**：
- `family`：协议族（AF_INET, AF_INET6, AF_UNIX 等）
- `type`：套接字类型（SOCK_STREAM, SOCK_DGRAM 等）
- `protocol`：具体协议（通常为 0，自动选择）

**返回值**：
- 成功：文件描述符（套接字）
- 失败：-1

#### 实现流程

**位置**：`../common/net/socket.c`

**关键步骤**：

1. **创建套接字**
   ```c
   sock_create(family, type, protocol, &sock);
   ```

2. **分配文件描述符**
   ```c
   fd = sock_map_fd(sock, flags & O_CLOEXEC);
   ```

3. **返回文件描述符**
   ```c
   return fd;
   ```

#### 套接字类型

**SOCK_STREAM（TCP）**：
- 面向连接的可靠传输
- 保证数据顺序和完整性

**SOCK_DGRAM（UDP）**：
- 无连接的数据报传输
- 不保证顺序和可靠性

### connect 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(connect, int, fd, struct sockaddr __user *, uservaddr, int, addrlen)
```

**功能**：
- 建立到远程服务器的连接
- 用于客户端

#### 实现流程

**位置**：`../common/net/socket.c`

**关键步骤**：

1. **获取套接字**
   ```c
   struct socket *sock = sockfd_lookup_light(fd, &err, &fput_needed);
   ```

2. **复制地址**
   ```c
   struct sockaddr_storage address;
   err = move_addr_to_kernel(uservaddr, addrlen, &address);
   ```

3. **建立连接**
   ```c
   err = sock->ops->connect(sock, (struct sockaddr *)&address, addrlen, sock->file->f_flags);
   ```

### bind 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(bind, int, fd, struct sockaddr __user *, umyaddr, int, addrlen)
```

**功能**：
- 绑定本地地址
- 用于服务器端

### listen 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE2(listen, int, fd, int, backlog)
```

**功能**：
- 开始监听连接
- 设置连接队列大小

### accept 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(accept, int, fd, struct sockaddr __user *, upeer_sockaddr, int __user *, upeer_addrlen)
```

**功能**：
- 接受客户端连接
- 返回新的套接字文件描述符

### send/recv 系统调用详解

#### send 函数签名

```c
SYSCALL_DEFINE4(send, int, fd, void __user *, buff, size_t, len, unsigned int, flags)
```

**功能**：
- 发送数据到连接的套接字
- TCP 专用

#### recv 函数签名

```c
SYSCALL_DEFINE4(recv, int, fd, void __user *, buff, size_t, len, unsigned int, flags)
```

**功能**：
- 从连接的套接字接收数据
- TCP 专用

### sendto/recvfrom 系统调用详解

#### sendto 函数签名

```c
SYSCALL_DEFINE6(sendto, int, fd, void __user *, buff, size_t, len, unsigned int, flags, struct sockaddr __user *, addr, int, addrlen)
```

**功能**：
- 发送数据到指定地址
- UDP 专用

#### recvfrom 函数签名

```c
SYSCALL_DEFINE6(recvfrom, int, fd, void __user *, buff, size_t, len, unsigned int, flags, struct sockaddr __user *, addr, int __user *, addrlen)
```

**功能**：
- 从套接字接收数据并获取发送者地址
- UDP 专用

## 总结

### 核心要点

1. **socket 创建套接字**：返回文件描述符
2. **connect 建立连接**：客户端使用
3. **bind/listen/accept**：服务器端使用
4. **send/recv 传输数据**：TCP 数据传输

### 关键系统调用

- **socket**：创建套接字
- **connect**：建立连接
- **bind/listen/accept**：服务器端操作
- **send/recv**：数据传输
