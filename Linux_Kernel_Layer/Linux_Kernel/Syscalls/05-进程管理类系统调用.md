# 进程管理类系统调用

## 学习目标

- 理解进程管理类系统调用的作用
- 掌握 fork/clone 系统调用的实现
- 理解 execve 系统调用的机制
- 了解 exit/wait 系统调用
- 掌握进程 ID 管理相关系统调用
- 理解进程创建和销毁的完整流程

## 背景介绍

进程管理是操作系统的核心功能之一。理解进程管理类系统调用的实现，有助于理解进程生命周期、进程间通信，以及系统资源管理。

## 核心概念

### 进程管理类系统调用分类

#### 1. 进程创建

- **fork()** - 创建子进程（传统方式）
- **clone()** - 创建进程/线程（更灵活）
- **clone3()** - clone 的扩展版本

#### 2. 进程执行

- **execve()** - 执行程序
- **execveat()** - 在指定目录执行程序

#### 3. 进程终止

- **exit()** - 正常退出
- **_exit()** - 立即退出（不清理）

#### 4. 进程等待

- **wait()** - 等待任意子进程
- **waitpid()** - 等待指定子进程
- **wait4()** - waitpid 的扩展版本

#### 5. 进程信息

- **getpid()** - 获取进程 ID
- **getppid()** - 获取父进程 ID
- **getuid()** / **geteuid()** - 获取用户 ID
- **getgid()** / **getegid()** - 获取组 ID

### fork 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE0(fork)
```

**功能**：
- 创建当前进程的副本（子进程）
- 子进程是父进程的完整副本
- 父子进程共享代码段，但数据段独立

**返回值**：
- 父进程：子进程的 PID
- 子进程：0
- 失败：-1

#### 实现流程

**位置**：`../common/kernel/fork.c`

**关键步骤**：

1. **调用 _do_fork**
   ```c
   return _do_fork(SIGCHLD, 0, 0, NULL, NULL, 0);
   ```

2. **_do_fork 函数**
   - 分配新的进程描述符
   - 复制父进程的资源
   - 设置子进程的初始状态
   - 唤醒子进程

#### 写时复制（COW）

**机制**：
- 父子进程初始共享物理页面
- 只有当进程修改页面时才复制
- 节省内存，提高性能

**实现**：
- 页表项标记为只读
- 写操作触发页面错误
- 页面错误处理程序复制页面

### clone 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE5(clone, unsigned long, clone_flags, unsigned long, newsp,
                 int __user *, parent_tidptr, unsigned long, tls,
                 int __user *, child_tidptr)
```

**功能**：
- 更灵活的进程/线程创建
- 可以控制共享的资源
- 可以创建线程（共享地址空间）

**参数**：
- `clone_flags`：控制共享的资源
- `newsp`：新栈指针
- `parent_tidptr`：父进程线程 ID 指针
- `tls`：线程本地存储
- `child_tidptr`：子进程线程 ID 指针

#### clone_flags

**常用标志**：
- `CLONE_VM`：共享地址空间（线程）
- `CLONE_FILES`：共享文件描述符表
- `CLONE_FS`：共享文件系统信息
- `CLONE_SIGHAND`：共享信号处理
- `SIGCHLD`：子进程退出时发送信号

### execve 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(execve, const char __user *, filename,
                const char __user *const __user *, argv,
                const char __user *const __user *, envp)
```

**功能**：
- 执行新的程序
- 替换当前进程的映像
- 保留进程 ID，但代码和数据完全替换

**参数**：
- `filename`：可执行文件路径
- `argv`：命令行参数数组
- `envp`：环境变量数组

#### 实现流程

**位置**：`../common/fs/exec.c`

**关键步骤**：

1. **加载可执行文件**
   ```c
   struct linux_binprm *bprm;
   bprm = alloc_bprm(fd, filename);
   ```

2. **解析可执行文件格式**
   - ELF 格式
   - 脚本格式
   - 其他格式

3. **替换进程映像**
   - 释放旧的内存映射
   - 加载新的代码和数据
   - 设置新的入口点

4. **启动新程序**
   - 跳转到新程序的入口点
   - 开始执行新程序

### exit 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE1(exit, int, error_code)
```

**功能**：
- 终止当前进程
- 清理进程资源
- 通知父进程

#### 实现流程

**位置**：`../common/kernel/exit.c`

**关键步骤**：

1. **设置退出状态**
   ```c
   do_exit((error_code & 0xff) << 8);
   ```

2. **do_exit 函数**
   - 释放进程资源
   - 关闭文件描述符
   - 发送信号给父进程
   - 将进程状态设置为僵尸进程

3. **进程清理**
   - 释放内存
   - 释放文件系统资源
   - 释放信号资源

### wait 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(wait4, pid_t, upid, int __user *, stat_addr,
                int, options, struct rusage __user *, ru)
```

**功能**：
- 等待子进程退出
- 获取子进程的退出状态
- 清理僵尸进程

**参数**：
- `upid`：要等待的子进程 PID（-1 表示任意子进程）
- `stat_addr`：退出状态存储地址
- `options`：选项（WNOHANG 等）
- `ru`：资源使用情况

#### 实现流程

**位置**：`../common/kernel/exit.c`

**关键步骤**：

1. **查找子进程**
   ```c
   struct pid *pid;
   pid = find_get_pid(upid);
   ```

2. **等待子进程退出**
   - 如果子进程已退出，立即返回
   - 如果子进程未退出，阻塞等待

3. **清理僵尸进程**
   - 释放进程描述符
   - 返回退出状态

### 进程 ID 管理

#### getpid 系统调用

**函数签名**：
```c
SYSCALL_DEFINE0(getpid)
```

**功能**：
- 返回当前进程的 PID
- 通过 vDSO 优化（避免系统调用）

#### getppid 系统调用

**函数签名**：
```c
SYSCALL_DEFINE0(getppid)
```

**功能**：
- 返回父进程的 PID

### 进程创建和销毁的完整流程

#### 进程创建流程

```
用户程序调用 fork()
    ↓
系统调用入口
    ↓
_do_fork()
    ↓
copy_process() - 复制进程描述符
    ↓
copy_mm() - 复制内存空间（COW）
    ↓
copy_files() - 复制文件描述符
    ↓
copy_signal() - 复制信号处理
    ↓
wake_up_new_task() - 唤醒新进程
    ↓
返回子进程 PID
```

#### 进程销毁流程

```
用户程序调用 exit()
    ↓
系统调用入口
    ↓
do_exit()
    ↓
释放资源
    ↓
发送 SIGCHLD 信号给父进程
    ↓
将进程状态设置为僵尸
    ↓
调度其他进程
```

## 实际应用

### 追踪进程创建

```bash
# 追踪 fork 和 execve
strace -e trace=fork,clone,execve <application>

# 查看进程树
pstree -p
```

### 分析进程性能

```bash
# 统计进程相关系统调用
strace -c -e trace=fork,execve,exit,wait <application>
```

## 与你的性能问题的关联

进程管理类系统调用可能与你的性能问题相关：

1. **进程创建开销**：fork/execve 的开销
2. **写时复制**：COW 机制的性能影响
3. **进程切换**：频繁的进程切换开销
4. **僵尸进程**：未正确 wait 导致的资源泄漏

**分析方法**：
```bash
# 统计 GO 应用的进程操作
strace -c -e trace=fork,execve,exit,wait <go_application>
```

## 总结

### 核心要点

1. **fork 创建子进程**：通过写时复制机制节省内存
2. **execve 执行程序**：替换进程映像
3. **exit 终止进程**：清理资源，通知父进程
4. **wait 等待子进程**：清理僵尸进程

### 关键系统调用

- **fork/clone**：创建进程/线程
- **execve**：执行程序
- **exit**：终止进程
- **wait/waitpid**：等待子进程
- **getpid/getppid**：获取进程 ID
