# 信号和同步类系统调用

## 学习目标

- 理解信号处理类系统调用的作用
- 掌握 signal/sigaction 系统调用的实现
- 理解 kill/tgkill 等信号发送系统调用
- 了解 futex 快速用户空间互斥锁
- 掌握信号传递机制和同步原语

## 背景介绍

信号是进程间通信的重要机制，同步原语是并发编程的基础。理解这些系统调用有助于分析进程间通信、并发控制，以及系统行为。

## 核心概念

### 信号处理类系统调用

#### 1. 信号注册

- **signal()** - 注册信号处理函数（简单接口）
- **sigaction()** - 注册信号处理函数（完整接口）
- **rt_sigaction()** - 实时信号处理

#### 2. 信号发送

- **kill()** - 发送信号给进程
- **tgkill()** - 发送信号给线程
- **tkill()** - 发送信号给线程（已废弃）

#### 3. 信号管理

- **sigprocmask()** - 设置信号掩码
- **sigsuspend()** - 等待信号
- **sigpending()** - 获取待处理信号

### signal 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE2(signal, int, sig, __sighandler_t, handler)
```

**功能**：
- 注册信号处理函数
- 简化接口，功能有限

### sigaction 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(rt_sigaction, int, sig, const struct sigaction __user *, act, struct sigaction __user *, oact, size_t, sigsetsize)
```

**功能**：
- 注册信号处理函数
- 提供完整的信号处理控制

### kill 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE2(kill, pid_t, pid, int, sig)
```

**功能**：
- 发送信号给进程或进程组

### futex 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE6(futex, u32 __user *, uaddr, int, op, u32, val, const struct __kernel_timespec __user *, utime, u32 __user *, uaddr2, u32, val3)
```

**功能**：
- 快速用户空间互斥锁
- 用于实现同步原语
- 性能优化的同步机制


## 总结

### 核心要点

1. **信号是进程间通信机制**：用于通知进程事件
2. **futex 是高性能同步原语**：用于实现锁和条件变量
3. **信号处理需要谨慎**：避免信号处理函数中的阻塞操作

### 关键系统调用

- **signal/sigaction**：注册信号处理
- **kill/tgkill**：发送信号
- **futex**：快速用户空间互斥锁
