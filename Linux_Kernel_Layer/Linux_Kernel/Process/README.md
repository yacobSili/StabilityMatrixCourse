# 进程管理系列文章（共 19 篇）

## 系列概述

本系列从架构师视角全面介绍 Linux/Android 进程管理，涵盖 Framework 层和 Kernel 层，详细阐述进程的上下游模块关系、核心机制和 Android 特有实现。

### 设计原则

1. **全局架构视角**：第一篇建立 Framework → Kernel 的完整视图
2. **模块关系清晰**：明确上下游、平级交互、无直接联系的模块
3. **生命周期导向**：从创建到退出的完整流程
4. **Android 特化**：专门章节介绍 Android 进程模型

## 模块关系总图

```
                          Framework 层
┌─────────────────────────────────────────────────────────────┐
│   ActivityManagerService │ ProcessList │ Zygote │ App      │
└────────────────────────────┬────────────────────────────────┘
                             │ 系统调用
┌────────────────────────────▼────────────────────────────────┐
│                     系统调用入口                             │
│         fork │ clone │ execve │ exit │ wait                │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                   进程管理子系统                             │
│              kernel/fork.c │ kernel/exit.c                  │
│                    task_struct                              │
└────────────────────────────┬────────────────────────────────┘
          │                  │                  │
    ┌─────▼─────┐     ┌─────▼─────┐     ┌─────▼─────┐
    │ 调度子系统 │     │ 内存管理  │     │ 文件系统  │
    │ kernel/   │     │ mm/       │     │ fs/       │
    │ sched/    │     │ mm_struct │     │files_struct│
    └─────┬─────┘     └───────────┘     └───────────┘
          │
    ┌─────▼─────┐
    │ 信号子系统 │
    │ kernel/   │
    │ signal.c  │
    └───────────┘
```

## 核心数据结构关系

```
task_struct（进程描述符）
    │
    ├── mm_struct ───────► 进程地址空间（与内存管理交互）
    │
    ├── files_struct ────► 打开文件表（与文件系统交互）
    │
    ├── fs_struct ───────► 文件系统信息（根目录、当前目录）
    │
    ├── signal_struct ───► 信号处理（与信号子系统交互）
    │
    ├── thread_info ─────► 线程信息（与调度器交互）
    │
    └── nsproxy ─────────► 命名空间（容器化支持）
```
