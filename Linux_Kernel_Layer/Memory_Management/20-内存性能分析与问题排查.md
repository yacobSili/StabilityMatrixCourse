## 三、常用工具

### 3.1 free

```bash
$ free -h
              total       used       free     shared    buff/cache   available
Mem:          7.8Gi      3.4Gi      1.2Gi       256Mi         3.2Gi        3.9Gi
Swap:         2.0Gi         0B      2.0Gi

# -h: 人类可读格式
# -w: 分开显示 buffers 和 cache
# -s N: 每 N 秒刷新
```

### 3.2 vmstat

```bash
$ vmstat 1
procs -----------memory---------- ---swap-- -----io---- -system-- ------cpu-----
 r  b   swpd   free   buff  cache   si   so    bi    bo   in   cs us sy id wa st
 1  0      0 123456  12345 234567    0    0     1     2  100  200  5  2 93  0  0

# r: 运行队列长度
# b: 阻塞进程数
# swpd: 使用的 swap
# free: 空闲内存
# buff: buffer cache
# cache: page cache
# si/so: swap in/out
# bi/bo: block in/out
```

### 3.3 top/htop

```bash
$ top -p <pid>
# 按 M 键按内存排序

# 关键字段
# VIRT: 虚拟内存
# RES: 驻留内存
# SHR: 共享内存
# %MEM: 内存占比
```

### 3.4 pmap

```bash
$ pmap -x <pid>

0000000000452000     184     184       0 r-x-- app
...
----------------  ------  ------  ------
total kB          123456   45678    1234
```

### 3.5 smem


## 五、内存泄漏检测

### 5.1 使用 LeakCanary (Android)

```groovy
// build.gradle
dependencies {
    debugImplementation 'com.squareup.leakcanary:leakcanary-android:2.12'
}

// 自动检测 Activity、Fragment、View 等泄漏
```

### 5.2 使用 malloc_debug

```bash
# 启用 malloc 调试
$ adb shell setprop libc.debug.malloc.options "backtrace leak_track"

# 重启应用
$ adb shell am force-stop <package>
$ adb shell am start -n <package>/<activity>

# 查看泄漏报告
$ adb shell dumpsys meminfo <pid> -d

# 或导出 malloc 信息
$ adb shell am dumpheap -n <pid> /data/local/tmp/heap.txt
```

### 5.3 使用 heapprofd (Android 10+)

```bash
# 使用 Perfetto 采集堆信息
$ adb shell perfetto --out /data/misc/perfetto-traces/heap.pb \
    --txt --config - <<EOF
data_sources {
    config {
        name: "android.heapprofd"
        heapprofd_config {
            process_cmdline: "com.example.app"
            sampling_interval_bytes: 4096
            continuous_dump_config {
                dump_phase_ms: 10000
                dump_interval_ms: 1000
            }
        }
    }
}
duration_ms: 30000
EOF

# 下载并分析
$ adb pull /data/misc/perfetto-traces/heap.pb
$ perfetto --open heap.pb
```

### 5.4 Native 内存泄漏

```bash
# 使用 AddressSanitizer (ASan)
# 编译时启用
LOCAL_SANITIZE := address

# 或运行时启用（需要支持的设备）
$ adb shell setprop wrap.com.example.app "ASAN_OPTIONS=detect_leaks=1"

# 使用 valgrind（Linux）
$ valgrind --leak-check=full ./app
```

## 六、常见问题排查

### 6.1 内存不足排查

```bash
# 1. 查看系统内存状态
$ cat /proc/meminfo | head -10
$ cat /proc/pressure/memory

# 2. 查看 OOM 日志
$ dmesg | grep -i oom

# 3. 查看进程内存使用
$ ps aux --sort=-%mem | head -20

# 4. 查看 LMKD 日志 (Android)
$ adb logcat -s lmkd:*

# 5. 检查 swap 使用
$ free -m
$ cat /proc/swaps
```

### 6.2 内存泄漏排查

```bash
# 1. 监控进程内存增长
$ while true; do
    cat /proc/<pid>/status | grep VmRSS
    sleep 5
done

# 2. 对比 smaps 差异
$ cat /proc/<pid>/smaps > smaps_t1.txt
# 运行一段时间后
$ cat /proc/<pid>/smaps > smaps_t2.txt
$ diff smaps_t1.txt smaps_t2.txt

# 3. 使用 pmap 监控
$ pmap -x <pid> > pmap_t1.txt
# 运行一段时间后
$ pmap -x <pid> > pmap_t2.txt
$ diff pmap_t1.txt pmap_t2.txt
```

### 6.3 碎片问题排查

```bash
# 1. 查看伙伴系统状态
$ cat /proc/buddyinfo
# 检查高 order 空闲块是否很少

# 2. 查看压缩统计
$ cat /proc/vmstat | grep compact

# 3. 触发内存压缩
$ echo 1 > /proc/sys/vm/compact_memory

# 4. 查看碎片指数
$ cat /sys/kernel/debug/extfrag/extfrag_index
```

### 6.4 Page Cache 问题

```bash
# 1. 查看 Page Cache 使用
$ cat /proc/meminfo | grep -E "Cached|Dirty|Writeback"

# 2. 监控脏页
$ while true; do
    grep -E "Dirty|Writeback" /proc/meminfo
    sleep 1
done

# 3. 强制回写
$ sync

# 4. 清空 Page Cache（谨慎使用）
$ echo 3 > /proc/sys/vm/drop_caches
```

## 七、性能调优建议

### 7.1 系统级调优

```bash
# 调整 swappiness
$ echo 60 > /proc/sys/vm/swappiness

# 调整脏页回写
$ echo 10 > /proc/sys/vm/dirty_background_ratio
$ echo 20 > /proc/sys/vm/dirty_ratio

# 调整最小空闲内存
$ echo 65536 > /proc/sys/vm/min_free_kbytes

# 调整 VFS 缓存压力
$ echo 100 > /proc/sys/vm/vfs_cache_pressure
```

### 7.2 应用级优化

```java
// Android 应用
// 1. 响应内存回调
@Override
public void onTrimMemory(int level) {
    if (level >= TRIM_MEMORY_MODERATE) {
        // 释放非关键缓存
        imageCache.clear();
    }
}

// 2. 避免内存泄漏
// - 避免 static 引用 Activity
// - 及时注销监听器
// - 使用 WeakReference

// 3. 优化 Bitmap
BitmapFactory.Options options = new BitmapFactory.Options();
options.inSampleSize = 2;  // 降采样
options.inBitmap = reusableBitmap;  // 复用
```

## 总结

### 关键工具

| 工具 | 用途 |
|-----|------|
| /proc/meminfo | 系统内存概览 |
| /proc/vmstat | 内存事件统计 |
| /proc/pid/smaps | 进程内存详情 |
| free/vmstat | 实时监控 |
| dumpsys meminfo | Android 内存报告 |
| heapprofd | Native 堆分析 |
| LeakCanary | Java 泄漏检测 |

### 关键指标

| 指标 | 说明 | 关注场景 |
|-----|------|---------|
| MemAvailable | 可用内存 | 系统整体状态 |
| PSS | 比例共享内存 | 进程内存占用 |
| Private_Dirty | 私有脏页 | 实际内存消耗 |
| pgmajfault | 主缺页 | I/O 性能 |
| pgscan_direct | 直接回收扫描 | 内存压力 |

### 问题排查流程

1. **确定问题类型**：泄漏、不足、碎片、性能
2. **收集信息**：meminfo、vmstat、进程 smaps
3. **分析原因**：比对、趋势、热点
4. **验证修复**：监控、测试、回归
