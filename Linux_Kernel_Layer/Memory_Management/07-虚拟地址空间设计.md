# 虚拟地址空间设计

## 学习目标

- 理解 Linux 虚拟地址空间的整体布局
- 掌握用户空间和内核空间的划分
- 了解 ARM64 和 x86_64 架构的地址空间特点
- 理解 ASLR 等安全机制

## 一、虚拟地址空间概述

### 1.1 为什么需要虚拟地址

1. **内存隔离**：每个进程有独立的地址空间，互不干扰
2. **内存保护**：通过页表权限控制访问
3. **简化编程**：进程看到连续的地址空间
4. **内存共享**：多个虚拟地址可映射同一物理页
5. **按需分配**：虚拟地址可以不立即对应物理内存

### 1.2 地址空间划分

```
64位虚拟地址空间（以 ARM64 为例）：

0xFFFF_FFFF_FFFF_FFFF ┌─────────────────────────┐
                      │                         │
                      │      内核空间           │
                      │   (256TB, 高地址)       │
                      │                         │
0xFFFF_0000_0000_0000 ├─────────────────────────┤
                      │                         │
                      │    非规范地址空洞        │
                      │   (用于地址校验)        │
                      │                         │
0x0000_FFFF_FFFF_FFFF ├─────────────────────────┤
                      │                         │
                      │      用户空间           │
                      │   (256TB, 低地址)       │
                      │                         │
0x0000_0000_0000_0000 └─────────────────────────┘
```

## 二、用户空间布局

### 2.1 典型进程地址空间

```
高地址
┌─────────────────────────────────────┐
│           内核空间                   │ ← 用户不可访问
├─────────────────────────────────────┤ ← TASK_SIZE
│           栈 (Stack)                │ ← 向下增长
│              ↓                      │
│                                     │
│           随机间隙                   │ ← ASLR
│                                     │
│              ↑                      │
│           mmap 区域                  │ ← 向上增长（或向下）
│   (共享库、文件映射、匿名映射)        │
│                                     │
│           随机间隙                   │ ← ASLR
│                                     │
│              ↑                      │
│           堆 (Heap)                 │ ← brk 向上扩展
├─────────────────────────────────────┤ ← start_brk
│           BSS 段                    │ ← 未初始化数据
├─────────────────────────────────────┤
│           数据段 (Data)             │ ← 已初始化数据
├─────────────────────────────────────┤
│           代码段 (Text)             │ ← 可执行代码
├─────────────────────────────────────┤
│           ELF 头                    │
├─────────────────────────────────────┤
│           保留区域                   │ ← NULL 指针保护
└─────────────────────────────────────┘
低地址
```

### 2.4 mmap 布局模式

```c
// 两种布局模式

// 传统布局（Legacy Layout）
// mmap 从低地址向高地址增长
//
// 高地址 ┌─────────┐
//        │  栈     │ ↓
//        │         │
//        │ mmap ↑  │
//        │         │
//        │ 堆 ↑    │
// 低地址 └─────────┘

// 现代布局（Top-Down Layout）- 默认
// mmap 从高地址向低地址增长
//
// 高地址 ┌─────────┐
//        │  栈     │ ↓
//        │         │
//        │ mmap ↓  │
//        │         │
//        │ 堆 ↑    │
// 低地址 └─────────┘

// arch/arm64/mm/mmap.c
unsigned long arch_get_unmapped_area_topdown(struct file *filp,
                                             const unsigned long addr0,
                                             const unsigned long len,
                                             const unsigned long pgoff,
                                             const unsigned long flags)
{
    struct vm_area_struct *vma;
    struct mm_struct *mm = current->mm;
    unsigned long addr = addr0;
    
    // 从高地址向下搜索空闲区域
    info.low_limit = PAGE_SIZE;
    info.high_limit = mm->mmap_base;
    info.length = len;
    
    addr = vm_unmapped_area(&info);
    
    return addr;
}
```

## 三、内核空间布局

### 3.1 ARM64 内核空间布局

```
内核虚拟地址空间 (ARM64, 48位虚拟地址)：

0xFFFF_FFFF_FFFF_FFFF ┌─────────────────────────┐
                      │   固定映射区            │ ← 早期启动映射
0xFFFF_FEFF_FFFF_FFFF ├─────────────────────────┤
                      │   PCI I/O 空间          │
0xFFFF_FDFF_FFFF_FFFF ├─────────────────────────┤
                      │   vmemmap               │ ← struct page 数组
0xFFFF_FCFF_FFFF_FFFF ├─────────────────────────┤
                      │   vmalloc 区域          │ ← vmalloc 分配
0xFFFF_0000_0000_0000 ├─────────────────────────┤
                      │   线性映射区            │ ← 直接映射所有物理内存
                      │   (物理内存线性映射)     │
                      │   PAGE_OFFSET           │
0xFFFF_8000_0000_0000 └─────────────────────────┘
```

### 3.2 关键常量定义

```c
// arch/arm64/include/asm/memory.h

// 用户空间大小（48位地址）
#define TASK_SIZE       (UL(1) << VA_BITS)     // 0x0001_0000_0000_0000 (256TB)

// 内核空间起始
#define PAGE_OFFSET     (UL(0xFFFF800000000000)) // -128TB

// vmalloc 区域
#define VMALLOC_START   (UL(0xFFFF000000000000))
#define VMALLOC_END     (PAGE_OFFSET - 1)

// vmemmap 区域
#define VMEMMAP_START   (UL(0xFFFF_FC00_0000_0000))

// 模块区域
#define MODULES_VADDR   (UL(0xFFFF800008000000))
#define MODULES_END     (UL(0xFFFF80007FFFFFFF))

// 内核镜像
#define KIMAGE_VADDR    (UL(0xFFFF800010000000))
```

### 3.3 内核地址类型

```c
// 1. 线性映射地址（直接映射）
// 物理地址 + PAGE_OFFSET = 虚拟地址

{
    return (void *)__phys_to_virt(x);

{
    return __virt_to_phys((unsigned long)(x));
```

## 四、ARM64 vs x86_64 地址空间

### 4.1 地址空间大小对比

| 特性 | ARM64 (48-bit VA) | x86_64 (48-bit VA) |
|-----|-------------------|-------------------|
| 用户空间 | 0 ~ 256TB | 0 ~ 128TB |
| 空洞 | 中间 | 中间 |
| 内核空间 | -256TB ~ 0 | -128TB ~ 0 |
| 页大小 | 4KB/16KB/64KB | 4KB |
| 大页 | 2MB/1GB | 2MB/1GB |

### 4.2 x86_64 内核空间布局

```
x86_64 内核虚拟地址空间：

0xFFFF_FFFF_FFFF_FFFF ┌─────────────────────────┐
                      │   未使用               │
0xFFFF_FFFF_8000_0000 ├─────────────────────────┤
                      │   内核代码（-2GB）      │
0xFFFF_FFFF_0000_0000 ├─────────────────────────┤
                      │   模块空间             │
0xFFFF_C900_0000_0000 ├─────────────────────────┤
                      │   vmalloc/ioremap      │
0xFFFF_C880_0000_0000 ├─────────────────────────┤
                      │   vmemmap              │
0xFFFF_8800_0000_0000 ├─────────────────────────┤
                      │   直接映射区           │
                      │   (最大 64TB)          │
0xFFFF_8000_0000_0000 └─────────────────────────┘
```

### 4.3 地址转换宏对比

```c
// ARM64
#define __va(x)     ((void *)((unsigned long)(x) + PAGE_OFFSET))
#define __pa(x)     ((unsigned long)(x) - PAGE_OFFSET)

// x86_64
#define __va(x)     ((void *)((unsigned long)(x) + PAGE_OFFSET))
#define __pa(x)     ((unsigned long)(x) - PAGE_OFFSET)

// 两者概念相同，但 PAGE_OFFSET 值不同
// ARM64: 0xFFFF_8000_0000_0000
// x86_64: 0xFFFF_8880_0000_0000
```

## 五、地址空间安全机制

### 5.1 ASLR (地址空间布局随机化)

```c
// kernel/fork.c
// 进程创建时随机化地址空间

static int setup_randomization(struct mm_struct *mm)
{
    unsigned long random_factor = 0;
    
    if (current->flags & PF_RANDOMIZE) {
        // 获取随机值
        random_factor = arch_mmap_rnd();
    }
    
    // 随机化 mmap 基址
    mm->mmap_base = mmap_base(random_factor);
    
    // 随机化栈
    mm->start_stack = STACK_TOP - random_factor;
    
    return 0;
}
```

**ASLR 随机化的区域**：

| 区域 | 随机化方式 |
|-----|----------|
| 栈 | 栈顶随机偏移 |
| mmap | mmap_base 随机 |
| 堆 | 可选随机化 |
| 可执行文件 | PIE 编译时 |
| 共享库 | 加载地址随机 |
| vDSO | 映射地址随机 |

### 5.2 栈保护

```c
// 栈增长保护
// VMA 标志包含 VM_GROWSDOWN

// 当访问栈外地址时：
// 1. 触发缺页异常
// 2. 检查是否在栈的增长范围内
// 3. 如果是，自动扩展栈
// 4. 否则，发送 SIGSEGV

static int expand_downwards(struct vm_area_struct *vma,
                           unsigned long address)
{
    // 检查栈大小限制
    if (size > rlimit(RLIMIT_STACK))
        return -ENOMEM;
    
    // 扩展 VMA
    vma->vm_start = address;
    vma->vm_pgoff = address >> PAGE_SHIFT;
    
    return 0;
}
```

### 5.4 KASLR (内核地址空间布局随机化)

```c
// 内核启动时随机化内核加载地址

// 随机化的部分：
// 1. 内核代码位置
// 2. 模块加载地址
// 3. 直接映射区域偏移

// arch/arm64/mm/kaslr.c
u64 __init kaslr_early_init(u64 dt_phys)
{
    u64 seed, offset;
    
    // 获取随机种子
    seed = get_kaslr_seed(dt_phys);
    
    // 计算随机偏移
    offset = (seed % (KASLR_OFFSET_MAX / KASLR_OFFSET_ALIGN)) * KASLR_OFFSET_ALIGN;
    
    return offset;
}
```

## 七、地址空间操作

### 7.1 进程创建时的地址空间

```c
// kernel/fork.c
static int dup_mm(struct task_struct *tsk, struct mm_struct *mm)
{
    // 复制 mm_struct
    struct mm_struct *new_mm = mm_alloc();
    
    // 复制所有 VMA
    retval = dup_mmap(new_mm, mm);
    
    // 复制页表（COW）
    retval = copy_page_range(new_mm, mm, vma);
    
    return 0;
}
```

### 7.2 exec 时的地址空间

```c
// fs/exec.c
static int exec_mmap(struct mm_struct *mm)
{
    struct mm_struct *old_mm;
    
    // 获取旧的 mm
    old_mm = current->mm;
    
    // 切换到新 mm
    current->mm = mm;
    current->active_mm = mm;
    
    // 激活新地址空间
    activate_mm(old_mm, mm);
    
    // 释放旧 mm
    if (old_mm) {
        mmput(old_mm);
    }
    
    return 0;
}
```

## 总结

### 核心概念

1. **用户空间**：低地址区域，每个进程独立
2. **内核空间**：高地址区域，所有进程共享
3. **线性映射**：物理内存的直接映射
4. **vmalloc**：虚拟连续但物理不连续的分配
5. **ASLR**：地址随机化安全机制

### 关键地址范围

| 区域 | ARM64 范围 |
|-----|-----------|
| 用户空间 | 0 ~ 0xFFFF_FFFF_FFFF |
| 内核线性映射 | 0xFFFF_8000_0000_0000 ~ |
| vmalloc | 0xFFFF_0000_0000_0000 ~ |
