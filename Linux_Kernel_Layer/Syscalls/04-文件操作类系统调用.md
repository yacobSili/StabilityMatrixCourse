# 文件操作类系统调用

## 学习目标

- 理解文件操作类系统调用的作用和分类
- 掌握 open/creat 系统调用的实现
- 理解 read/write 系统调用的机制
- 了解 close、lseek、ioctl 等系统调用
- 掌握文件描述符的管理
- 理解文件操作与 VFS 的关系

## 背景介绍

文件操作是系统调用中使用最频繁的一类。理解文件操作类系统调用的实现，有助于分析 I/O 性能问题、理解文件系统行为，以及优化文件操作。

## 核心概念

### 文件操作类系统调用分类

#### 1. 文件打开和创建

- **open()** - 打开文件/设备
- **creat()** - 创建文件（已废弃，使用 open 替代）
- **openat()** - 在指定目录打开文件

#### 2. 文件读写

- **read()** - 从文件读取数据
- **write()** - 向文件写入数据
- **pread()** / **pwrite()** - 指定位置读写
- **readv()** / **writev()** - 向量化读写

#### 3. 文件控制

- **close()** - 关闭文件
- **fcntl()** - 文件控制
- **ioctl()** - 设备控制
- **lseek()** - 文件定位

#### 4. 文件状态

- **stat()** / **fstat()** / **lstat()** - 获取文件状态
- **fstatfs()** - 获取文件系统状态

### open 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(open, const char __user *, filename, int, flags, umode_t, mode)
```

**参数**：
- `filename`：文件路径（用户空间指针）
- `flags`：打开标志（O_RDONLY, O_WRONLY, O_RDWR 等）
- `mode`：文件权限（创建文件时使用）

**返回值**：
- 成功：文件描述符（非负整数）
- 失败：-1，errno 设置错误码

#### 实现流程

**位置**：`../common/fs/open.c`

**关键步骤**：

1. **参数验证**
   ```c
   // 验证 flags
   if (force_o_largefile())
       flags |= O_LARGEFILE;
   ```

2. **路径解析**
   ```c
   // 解析文件路径
   struct filename *tmp = getname(filename);
   ```

3. **打开文件**
   ```c
   // 调用 do_sys_open
   fd = do_sys_open(AT_FDCWD, tmp->name, flags, mode);
   ```

4. **返回文件描述符**
   ```c
   putname(tmp);
   return fd;
   ```

#### do_sys_open 函数

**功能**：
- 分配文件描述符
- 打开文件
- 安装文件描述符

**关键代码**：
```c
long do_sys_open(int dfd, struct filename *name, int flags, umode_t mode)
{
    struct open_how how = build_open_how(flags, mode);
    return do_sys_openat2(dfd, name, &how);
}
```

#### 文件描述符分配

**机制**：
- 从进程的文件描述符表中分配
- 使用最小的可用文件描述符
- 文件描述符是进程私有的

**位置**：`../common/fs/file.c`

### read 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(read, unsigned int, fd, char __user *, buf, size_t, count)
```

**参数**：
- `fd`：文件描述符
- `buf`：缓冲区（用户空间指针）
- `count`：要读取的字节数

**返回值**：
- 成功：实际读取的字节数
- 失败：-1，errno 设置错误码

#### 实现流程

**位置**：`../common/fs/read_write.c`

**关键步骤**：

1. **获取文件对象**
   ```c
   struct fd f = fdget_pos(fd);
   ```

2. **读取数据**
   ```c
   if (f.file) {
       loff_t pos = file_pos_read(f.file);
       ret = vfs_read(f.file, buf, count, &pos);
       if (ret >= 0)
           file_pos_write(f.file, pos);
   }
   ```

3. **返回结果**
   ```c
   fdput_pos(f);
   return ret;
   ```

#### vfs_read 函数

**功能**：
- 通过 VFS 层读取文件
- 调用具体文件系统的 read 方法
- 处理用户空间指针

**关键代码**：
```c
ssize_t vfs_read(struct file *file, char __user *buf, size_t count, loff_t *pos)
{
    // 权限检查
    if (!(file->f_mode & FMODE_READ))
        return -EBADF;
    
    // 调用文件系统的 read 方法
    if (file->f_op->read)
        return file->f_op->read(file, buf, count, pos);
    else if (file->f_op->read_iter)
        return new_sync_read(file, buf, count, pos);
    else
        return -EINVAL;
}
```

### close 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE1(close, unsigned int, fd)
```

**功能**：
- 关闭文件描述符
- 释放相关资源
- 减少文件引用计数

#### 实现流程

**位置**：`../common/fs/open.c`

**关键步骤**：

1. **获取文件对象**
   ```c
   struct fd f = fdget(fd);
   ```

2. **关闭文件**
   ```c
   if (f.file) {
       filp_close(f.file, files);
       retval = 0;
   }
   ```

3. **释放文件描述符**
   ```c
   fdput(f);
   return retval;
   ```

### lseek 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(lseek, unsigned int, fd, off_t, offset, unsigned int, whence)
```

**功能**：
- 改变文件读写位置
- `whence`：SEEK_SET, SEEK_CUR, SEEK_END

**实现**：
- 通过 VFS 层调用文件系统的 llseek 方法

### ioctl 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(ioctl, unsigned int, fd, unsigned int, cmd, unsigned long, arg)
```

**功能**：
- 设备特定的控制操作
- 每个设备驱动可以定义自己的 ioctl 命令

**特点**：
- 非常灵活
- 设备特定
- 用于设备控制

### fcntl 系统调用详解

#### 函数签名

```c
SYSCALL_DEFINE3(fcntl, unsigned int, fd, unsigned int, cmd, unsigned long, arg)
```

**功能**：
- 文件控制操作
- 设置文件标志（O_NONBLOCK, O_APPEND 等）
- 文件锁定

### 文件描述符管理

#### 文件描述符表

**结构**：
- 每个进程有独立的文件描述符表
- 文件描述符是数组索引
- 数组元素指向文件对象

#### 文件对象

**结构**：
- `struct file`：表示打开的文件
- 包含文件操作函数指针
- 包含文件位置、标志等信息

#### 引用计数

**机制**：
- 文件对象有引用计数
- 多个文件描述符可以指向同一个文件对象
- close 时减少引用计数
- 引用计数为 0 时释放文件对象

## 实际应用

```bash

strace -e trace=open,read,write,close ls

# 查看文件描述符使用
strace -e trace=open,close ls 2>&1 | grep -E "open|close"
```

### 分析文件 I/O 性能

```bash
# 统计文件操作
strace -c -e trace=open,read,write,close <application>

# 查看特定文件的访问
strace -e trace=open,read,write <application> 2>&1 | grep "file.txt"
```

## 与你的性能问题的关联

文件操作类系统调用可能与你的性能问题相关：

1. **频繁的文件 I/O**：GO 应用可能频繁读写文件
2. **文件描述符泄漏**：未关闭的文件描述符
3. **I/O 阻塞**：同步 I/O 可能阻塞进程
4. **缓存效果**：文件系统缓存的影响

**分析方法**：
```bash
# 统计 GO 应用的文件操作
strace -c -e trace=open,read,write,close <go_application>

# 查看文件描述符使用
lsof -p <pid>
```

## 总结

### 核心要点

1. **文件操作是最常用的系统调用**：open, read, write, close
2. **通过 VFS 层访问文件系统**：统一的文件系统接口
3. **文件描述符管理**：每个进程有独立的文件描述符表
4. **文件对象引用计数**：多个描述符可以共享文件对象

### 关键系统调用

- **open**：打开文件，返回文件描述符
- **read**：读取文件数据
- **write**：写入文件数据
- **close**：关闭文件描述符
- **ioctl**：设备控制
- **fcntl**：文件控制
