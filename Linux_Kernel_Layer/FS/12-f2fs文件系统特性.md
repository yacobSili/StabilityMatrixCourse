# f2fs 文件系统特性

## 学习目标

- 理解 f2fs 的设计理念：闪存友好
- 掌握 f2fs 的日志结构与段管理
- 理解垃圾回收（GC）机制
- 了解 f2fs vs ext4 的对比
- 理解 f2fs 在 Android 中的应用与调优

## 概述

f2fs（Flash-Friendly File System）是专门为闪存设备设计的文件系统，在 Android 系统中广泛用于 /data 和 /cache 分区。

---

## 一、f2fs 设计理念

### 为什么需要 f2fs

传统文件系统（如 ext4）为机械硬盘设计，不适合闪存特性：
- **擦除块（Erase Block）**：闪存以块为单位擦除
- **写入放大（Write Amplification）**：频繁更新导致额外写入
- **磨损均衡（Wear Leveling）**：需要均匀使用闪存块

### f2fs 设计目标

1. **闪存友好**：考虑闪存特性
2. **性能优化**：顺序写入，减少碎片
3. **寿命延长**：减少写入放大
4. **快速恢复**：日志结构文件系统

---

## 二、f2fs 日志结构

### 日志结构文件系统（LFS）

f2fs 采用日志结构文件系统（Log-Structured File System）设计：
- 所有写入都是追加（append-only）
- 数据按顺序写入
- 旧数据通过垃圾回收清理

### f2fs 磁盘布局

```
f2fs 文件系统布局：

┌─────────────────────────────────────────────────────────────┐
│  Superblock (F2FS_SUPER_OFFSET)                             │
│  - 文件系统魔数                                              │
│  - 段数、节数                                                │
│  - CP（Checkpoint）位置                                      │
├─────────────────────────────────────────────────────────────┤
│  Checkpoint (CP)                                             │
│  - 文件系统状态                                              │
│  - 段使用位图                                                │
│  - 活动日志位置                                              │
├─────────────────────────────────────────────────────────────┤
│  Segment Summary Area (SSA)                                  │
│  - 段摘要信息                                                │
├─────────────────────────────────────────────────────────────┤
│  Main Area (数据区)                                          │
│  ┌──────────┬──────────┬──────────┬──────────┐            │
│  │  Segment │  Segment │  Segment │  Segment │            │
│  │    0     │    1     │    2     │   ...    │            │
│  └──────────┴──────────┴──────────┴──────────┘            │
│  每个 Segment 包含多个 Block                                │
└─────────────────────────────────────────────────────────────┘
```

### 段（Segment）管理

```c
// fs/f2fs/f2fs.h
// 段大小：通常 2MB（512 个 4KB 块）
#define F2FS_SEGMENT_SIZE(sbi)       (1 << (sbi)->log_blocks_per_seg)

// 段类型
enum {
    CURSEG_HOT_DATA = 0,    // 热数据段
    CURSEG_WARM_DATA,       // 温数据段
    CURSEG_COLD_DATA,       // 冷数据段
    CURSEG_HOT_NODE,        // 热节点段
    CURSEG_WARM_NODE,       // 温节点段
    CURSEG_COLD_NODE,       // 冷节点段
    NO_CHECK_TYPE,
};
```

## 三、垃圾回收（GC）机制

### GC 的作用

垃圾回收清理无效数据，回收空间，是 f2fs 的核心机制。

### GC 触发时机

1. **空间不足**：可用段数低于阈值
2. **后台 GC**：定期后台清理
3. **用户触发**：通过 ioctl 触发

### GC 流程

```c
// fs/f2fs/gc.c
int f2fs_gc(struct f2fs_sb_info *sbi, bool sync, bool background)
{
    unsigned int segno;
    int gc_type = sync ? FG_GC : BG_GC;
    int sec_freed = 0;
    
    // 1. 选择要回收的段
    segno = select_victim_seg(sbi, gc_type);
    if (segno == NULL_SEGNO)
        return 0;
    
    // 2. 执行垃圾回收
    sec_freed = do_garbage_collect(sbi, segno, gc_type);
    
    return sec_freed;
}
```

### 段选择策略

```c
// fs/f2fs/gc.c
static unsigned int select_victim_seg(struct f2fs_sb_info *sbi,
                                       int gc_type)
{
    // 1. 选择有效块最少的段
    // 2. 优先选择冷数据段
    // 3. 避免选择热数据段
    
    return victim_segno;
}
```

## 四、f2fs vs ext4 对比

### 性能对比

| 场景 | ext4 | f2fs | 说明 |
|-----|------|------|-----|
| 顺序写入 | 中 | 高 | f2fs 顺序写入优势 |
| 随机写入 | 中 | 中 | 两者相当 |
| 顺序读取 | 高 | 高 | 两者相当 |
| 随机读取 | 高 | 中 | ext4 略好 |
| 小文件 | 高 | 中 | ext4 元数据优化好 |
| 大文件 | 中 | 高 | f2fs extent 优势 |

### 特性对比

| 特性 | ext4 | f2fs |
|-----|------|------|
| 日志 | 支持 | 支持 |
| 延迟分配 | 支持 | 支持 |
| 在线扩展 | 支持 | 支持 |
| 压缩 | 不支持 | 支持 |
| 加密 | 支持 | 支持 |
| 闪存优化 | 否 | 是 |

### 使用建议

- **ext4**：适合混合存储、需要高随机读取性能
- **f2fs**：适合纯闪存、顺序写入多、需要延长闪存寿命

## 五、f2fs 在 Android 中的应用

### Android 分区使用

```bash
# 查看文件系统类型
mount | grep /data
# /dev/block/sda1 on /data type f2fs (rw,seclabel,relatime,background_gc=on)

# 查看 f2fs 信息
fsck.f2fs -l /dev/block/sda1
```

### f2fs 调优参数

```bash
# 调整 GC 策略
# /sys/fs/f2fs/<device>/gc_urgent
echo 1 > /sys/fs/f2fs/sda1/gc_urgent  # 紧急GC

# 调整后台 GC
# /sys/fs/f2fs/<device>/gc_idle
echo 1 > /sys/fs/f2fs/sda1/gc_idle    # 启用后台GC

# 查看段使用情况
cat /sys/fs/f2fs/sda1/segment_info
```

### Android 配置

```bash
# Android 启动脚本中配置 f2fs
# init.rc
mount f2fs /dev/block/sda1 /data \
    noatime,nosuid,nodev,background_gc=on,inline_xattr,inline_data
```

## 六、f2fs 关键机制

### 1. 多日志（Multi-head Logging）

f2fs 使用多个日志头，根据数据热度选择不同的日志：
- **热数据**：频繁更新的数据
- **温数据**：偶尔更新的数据
- **冷数据**：很少更新的数据

### 2. 节点结构

```c
// fs/f2fs/f2fs.h
// inode 节点
struct f2fs_inode {
    __le32 i_flags;              // 标志
    __le32 i_pino;               // 父inode号
    __le32 i_namelen;            // 文件名长度
    __u8 i_name[F2FS_NAME_LEN];   // 文件名
    __u8 i_reserved2[15];         // 保留
    struct f2fs_extent i_ext;     // extent信息
    __le32 i_addr[F2FS_ADDRS_PER_INODE]; // 地址数组
    // ...
};

// 直接节点（Direct Node）
struct f2fs_direct_node {
    __le32 addr[ADDRS_PER_BLOCK]; // 数据块地址
};

// 间接节点（Indirect Node）
struct f2fs_indirect_node {
    __le32 nid[NIDS_PER_BLOCK];   // 子节点ID
};
```

### 3. 检查点（Checkpoint）

f2fs 使用检查点机制保证一致性：
- 定期创建检查点
- 记录文件系统状态
- 崩溃后从检查点恢复
