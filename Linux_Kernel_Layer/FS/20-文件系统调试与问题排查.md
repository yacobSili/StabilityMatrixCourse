# 文件系统调试与问题排查

## 学习目标

- 掌握文件系统调试工具
- 了解 /proc 和 /sys 下的文件系统信息
- 理解文件系统损坏与修复
- 掌握 Android 文件系统相关问题排查
- 了解常见案例分析

## 概述

文件系统调试和问题排查是系统维护的重要技能。掌握调试工具、理解问题现象、快速定位问题是关键。

---

## 一、文件系统调试工具

### 1. mount 命令

```bash
# 查看挂载信息
mount | grep -E "ext4|f2fs|erofs"

# 查看挂载选项
mount -o remount,ro /data
```

### 2. df 命令

```bash
# 查看文件系统使用情况
df -h

# 输出示例
Filesystem      Size  Used Avail Use% Mounted on
/dev/block/sda5 64G   32G   32G  50% /data
```

### 3. du 命令

```bash
# 查看目录大小
du -sh /data/*

# 查找大文件
du -h /data | sort -rh | head -10
```

### 4. fsck 命令

```bash
# 检查 ext4 文件系统
fsck.ext4 -n /dev/block/sda5

# 检查 f2fs 文件系统
fsck.f2fs /dev/block/sda5
```

## 二、/proc 和 /sys 下的文件系统信息

### /proc 文件系统信息

```bash
# 文件系统统计
cat /proc/filesystems

# 挂载信息
cat /proc/mounts

# 文件系统统计
cat /proc/sys/fs/dentry-state
cat /proc/sys/fs/file-nr
```

### /sys 文件系统信息

```bash
# 块设备信息
ls /sys/block/

# 文件系统信息
cat /sys/fs/f2fs/sda1/segment_info
```

## 三、文件系统损坏与修复

### 损坏原因

1. **异常断电**：未完成的写入
2. **硬件故障**：存储设备问题
3. **软件 Bug**：内核或文件系统 Bug

### 修复方法

```bash
# 1. 卸载文件系统
umount /data

# 2. 检查文件系统
fsck.ext4 -f /dev/block/sda5

# 3. 修复（如果需要）
fsck.ext4 -y /dev/block/sda5

# 4. 重新挂载
mount /dev/block/sda5 /data
```

## 四、Android 文件系统相关问题排查

### 1. 存储空间不足

**排查步骤**：
```bash
# 查看空间使用
df -h

# 查找大文件
du -h /data | sort -rh | head -20

# 清理缓存
rm -rf /data/cache/*
```

### 2. 文件系统只读

**排查步骤**：
```bash
# 检查挂载选项
mount | grep /data

# 检查文件系统错误
dmesg | grep -i "ext4\|f2fs\|error"

# 检查 SELinux
getenforce
```

### 3. IO 性能问题

**排查步骤**：
```bash
# 查看 IO 统计
iostat -x 1

# 查看进程 IO
iotop

# 查看 Block 层统计
cat /proc/diskstats
```

## 五、常见案例分析

### 案例 1：文件系统损坏

**现象**：
- 文件系统变为只读
- 应用无法写入文件
- dmesg 显示文件系统错误

**排查**：
```bash
# 1. 查看 dmesg
dmesg | grep -i "ext4\|error"

# 2. 检查文件系统
fsck.ext4 -n /dev/block/sda5

# 3. 修复
fsck.ext4 -y /dev/block/sda5
```

### 案例 2：IO 性能下降

**现象**：
- 应用响应变慢
- IO 延迟增加

**排查**：
```bash
# 1. 查看 IO 统计
iostat -x 1

# 2. 查看进程 IO
iotop

# 3. 分析 Block 层
blktrace -d /dev/block/sda5
```
