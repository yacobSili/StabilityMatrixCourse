# 文件权限与安全机制

- 理解 Unix 文件权限模型（rwx）
- 掌握 Linux Capabilities

## 概述

文件权限和安全机制是文件系统的重要组成部分，确保文件访问的安全性和隔离性。

## 一、Unix 文件权限模型

### 权限位

```c
// 文件权限位
S_IRUSR  0400  // 用户读
S_IWUSR  0200  // 用户写
S_IXUSR  0100  // 用户执行
S_IRGRP  0040  // 组读
S_IWGRP  0020  // 组写
S_IXGRP  0010  // 组执行
S_IROTH  0004  // 其他读
S_IWOTH  0002  // 其他写
S_IXOTH  0001  // 其他执行
```

### 权限检查

```c
// fs/open.c
static int may_open(const struct path *path, int acc_mode, int flag)
{
    struct dentry *dentry = path->dentry;
    struct inode *inode = dentry->d_inode;
    int error;
    
    // 1. 检查文件是否存在
    if (!inode)
        return -ENOENT;
    
    // 2. 检查权限
    error = inode_permission(inode, acc_mode);
    if (error)
        return error;
    
    // 3. 检查打开标志
    if (flag & O_NOATIME && !inode_owner_or_capable(inode))
        return -EPERM;
    
    return 0;
}
```

## 二、Linux Capabilities

### Capabilities 概念

Capabilities 将 root 权限细分为多个能力，提供更精细的权限控制。

### 常见 Capabilities

| Capability | 说明 |
|-----------|------|
| CAP_DAC_OVERRIDE | 忽略文件权限 |
| CAP_SYS_ADMIN | 系统管理 |
| CAP_NET_ADMIN | 网络管理 |

## 三、SELinux 与文件标签

### SELinux 概念

SELinux（Security-Enhanced Linux）提供强制访问控制（MAC）。

### 文件标签

```bash
# 查看文件 SELinux 标签
ls -Z /data/file.txt
# -rw-r--r-- 1 u0_a100 u0_a100 u:object_r:app_data_file:s0 file.txt

# 标签格式
# u:object_r:app_data_file:s0
# │  │       │              │
# │  │       │              └─ MLS级别
# │  │       └─ 类型
# │  └─ 角色
# └─ 用户
```

### SELinux 策略

```bash
# SELinux 策略文件
/system/etc/selinux/plat_sepolicy.cil
/vendor/etc/selinux/vendor_sepolicy.cil
```

## 四、Android 文件沙箱机制

### 应用沙箱

每个 Android 应用运行在独立的沙箱中：
- 独立的用户 ID（UID）
- 独立的文件目录
- 独立的权限

### 文件访问隔离

```
应用 A (UID 10000)
/data/data/com.app.a/
    └── files/  # 只能应用 A 访问

应用 B (UID 10001)
/data/data/com.app.b/
    └── files/  # 只能应用 B 访问
```

## 五、跨应用文件共享

### 共享方式

1. **ContentProvider**：通过 URI 共享
2. **FileProvider**：安全的文件共享
3. **SAF**：用户选择文件

### FileProvider 使用

```xml
<!-- AndroidManifest.xml -->
<provider
    android:name="androidx.core.content.FileProvider"
    android:authorities="com.example.fileprovider"
    android:exported="false"
    android:grantUriPermissions="true">
    <meta-data
        android:name="android.support.FILE_PROVIDER_PATHS"
        android:resource="@xml/file_paths" />
</provider>
```
