# ext4 文件系统架构

## 学习目标

- 理解 ext4 文件系统的整体架构与设计理念
- 掌握 ext4 的磁盘布局：超级块、块组、inode 表
- 理解 extent 机制的工作原理
- 了解日志（journaling）机制
- 理解 ext4 在 Android 中的应用

## 概述

ext4（Fourth Extended File System）是 Linux 中最常用的文件系统之一，也是 Android 数据分区的主要文件系统。

---

## 一、ext4 设计理念

### 设计目标

1. **向后兼容**：兼容 ext2/ext3
2. **高性能**：优化大文件和小文件性能
3. **可扩展性**：支持大文件和大量文件
4. **可靠性**：日志机制保证数据一致性

### ext4 特性

| 特性 | 说明 | 优势 |
|-----|------|-----|
| Extent | 连续块管理 | 减少元数据，提高性能 |
| 日志 | 日志记录 | 快速恢复，数据一致性 |
| 延迟分配 | 延迟块分配 | 减少碎片，提高性能 |
| 多块分配 | 一次分配多个块 | 减少碎片 |
| 在线扩展 | 在线调整大小 | 无需卸载 |

### 超级块（Super Block）

**位置**：每个块组的第一个块（通常 Block 1）

**结构**：
```c
// fs/ext4/ext4.h
struct ext4_super_block {
    __le32  s_inodes_count;         // inode总数
    __le32  s_blocks_count_lo;      // 块总数
    __le32  s_r_blocks_count_lo;     // 保留块数
    __le32  s_free_blocks_count_lo; // 空闲块数
    __le32  s_free_inodes_count;    // 空闲inode数
    __le32  s_first_data_block;     // 第一个数据块
    __le32  s_log_block_size;       // 块大小（log2）
    __le32  s_log_cluster_size;      // 簇大小（log2）
    __le32  s_blocks_per_group;      // 每块组块数
    __le32  s_inodes_per_group;      // 每块组inode数
    __le32  s_mtime;                 // 挂载时间
    __le32  s_wtime;                 // 写入时间
    __le16  s_mnt_count;             // 挂载次数
    __le16  s_max_mnt_count;         // 最大挂载次数
    __le16  s_magic;                  // 魔数（0xEF53）
    __le16  s_state;                  // 文件系统状态
    __le16  s_errors;                 // 错误处理
    __le16  s_minor_rev_level;        // 次版本号
    __le32  s_lastcheck;              // 最后检查时间
    __le32  s_checkinterval;          // 检查间隔
    __le32  s_creator_os;             // 创建者OS
    __le32  s_rev_level;              // 版本号
    __le16  s_def_resuid;             // 默认保留UID
    __le16  s_def_resgid;             // 默认保留GID
    __le32  s_first_ino;              // 第一个inode号
    __le16  s_inode_size;             // inode大小
    __le16  s_block_group_nr;         // 块组号
    __le32  s_feature_compat;         // 兼容特性
    __le32  s_feature_incompat;        // 不兼容特性
    __le32  s_feature_ro_compat;       // 只读兼容特性
    __u8    s_uuid[16];               // UUID
    char    s_volume_name[16];        // 卷名
    char    s_last_mounted[64];       // 最后挂载点
    __le32  s_algorithm_usage_bitmap; // 压缩算法位图
    __u8    s_prealloc_blocks;        // 预分配块数
    __u8    s_prealloc_dir_blocks;    // 目录预分配块数
    __le16  s_reserved_gdt_blocks;    // 保留GDT块数
    __u8    s_journal_uuid[16];       // 日志UUID
    __le32  s_journal_inum;           // 日志inode号
    __le32  s_journal_dev;            // 日志设备号
    __le32  s_last_orphan;            // 最后孤儿inode
    __le32  s_hash_seed[4];           // 哈希种子
    __u8    s_def_hash_version;       // 默认哈希版本
    __u8    s_jnl_backup_type;        // 日志备份类型
    __le16  s_desc_size;              // 组描述符大小
    __le32  s_default_mount_opts;     // 默认挂载选项
    __le32  s_first_meta_bg;          // 第一个元数据块组
    __le32  s_mkfs_time;              // 创建时间
    __le32  s_jnl_blocks[17];         // 日志块
    // ... 更多字段
};
```

## 三、Extent 机制

### 传统块映射 vs Extent

#### 传统块映射（ext2/ext3）

```
小文件（12个直接块）：
inode
├── i_block[0] → 块 100
├── i_block[1] → 块 101
├── ...
└── i_block[11] → 块 111

大文件（间接块）：
inode
├── i_block[12] → 间接块
│   └── 间接块 → 块 200, 201, 202, ...
├── i_block[13] → 二级间接块
└── i_block[14] → 三级间接块
```

**问题**：
- 元数据开销大
- 随机访问性能差
- 碎片化严重

#### Extent 机制（ext4）

```
Extent 结构：
struct ext4_extent {
    __le32  ee_block;    // 逻辑块号（文件内偏移）
    __le16  ee_len;      // 连续块数
    __le32  ee_start_hi; // 起始物理块号（高32位）
    __le32  ee_start_lo; // 起始物理块号（低32位）
};

示例：
文件逻辑块 0-99 → 物理块 1000-1099（一个extent）
文件逻辑块 100-199 → 物理块 5000-5099（另一个extent）
```

**优势**：
- 元数据开销小
- 顺序访问性能好
- 减少碎片

### Extent 树

```c
// fs/ext4/ext4_extents.h
struct ext4_extent_header {
    __le16  eh_magic;       // 魔数（0xF30A）
    __le16  eh_entries;      // 条目数
    __le16  eh_max;          // 最大条目数
    __le16  eh_depth;        // 树深度
    __le32  eh_generation;   // 生成号
};

struct ext4_extent_idx {
    __le32  ei_block;        // 索引块号
    __le32  ei_leaf_lo;      // 叶子节点块号（低32位）
    __le16  ei_leaf_hi;      // 叶子节点块号（高16位）
    __u16   ei_unused;
};

// 小文件：直接extent（4个）
// 中等文件：单层索引（1个索引节点 + 多个extent）
// 大文件：多层索引（索引树）
```

## 四、日志（Journaling）机制

### 日志的作用

日志机制记录文件系统操作，在系统崩溃后可以快速恢复，保证数据一致性。

### 日志模式

| 模式 | 说明 | 性能 | 可靠性 |
|-----|------|------|-------|
| journal | 数据和元数据都记录 | 低 | 最高 |
| ordered | 只记录元数据，数据先写 | 中 | 高 |
| writeback | 只记录元数据 | 高 | 中 |

### 日志结构

```
日志布局：

┌─────────────────────────────────────────────────────────────┐
│  Journal Superblock (日志超级块)                             │
│  - 日志魔数                                                  │
│  - 日志块数                                                  │
│  - 日志序列号                                                │
├─────────────────────────────────────────────────────────────┤
│  Journal Block 1                                            │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Transaction 1                                       │   │
│  │  - 描述符块                                            │   │
│  │  - 数据块                                              │   │
│  │  - 提交块                                              │   │
│  └──────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  Journal Block 2                                            │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Transaction 2                                       │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 日志事务

```c
// fs/jbd2/transaction.c
handle_t *ext4_journal_start_sb(struct super_block *sb, int nblocks)
{
    journal_t *journal = EXT4_SB(sb)->s_journal;
    return jbd2_journal_start(journal, nblocks);
}

int ext4_journal_stop(handle_t *handle)
{
    return jbd2_journal_stop(handle);
}
```

## 五、ext4 在 Android 中的应用

### Android 分区使用

| 分区 | 文件系统 | 说明 |
|-----|---------|-----|
| /data | ext4 或 f2fs | 用户数据 |
| /cache | ext4 或 f2fs | 缓存数据 |
| /system | erofs | 系统分区（只读） |
| /vendor | erofs | 厂商分区（只读） |

### ext4 配置

```bash
# 查看 ext4 文件系统信息
tune2fs -l /dev/block/sda1

# 调整 ext4 参数
tune2fs -o journal_data /dev/block/sda1  # 启用数据日志
tune2fs -o ^has_journal /dev/block/sda1  # 禁用日志
```
