# 文件系统性能分析

## 学习目标

- 理解文件系统性能指标
- 掌握性能分析工具（iostat, blktrace, perf）
- 了解文件系统 benchmark（fio, iozone）
- 理解常见性能问题与优化

## 概述

文件系统性能分析是优化系统性能的关键。理解性能指标、使用分析工具、识别性能瓶颈是系统优化的基础。

---

## 一、文件系统性能指标

### 关键指标

| 指标 | 说明 | 单位 |
|-----|------|-----|
| IOPS | 每秒 IO 操作数 | ops/s |
| 吞吐量 | 每秒传输数据量 | MB/s |
| 延迟 | IO 操作响应时间 | ms |
| 队列深度 | 未完成的 IO 数 | 个 |

### 性能目标

| 场景 | IOPS | 吞吐量 | 延迟 |
|-----|------|-------|------|
| 随机读 | > 1000 | > 50 MB/s | < 10 ms |
| 顺序读 | > 5000 | > 200 MB/s | < 5 ms |
| 随机写 | > 500 | > 30 MB/s | < 20 ms |
| 顺序写 | > 3000 | > 150 MB/s | < 10 ms |

## 二、性能分析工具

### iostat

**作用**：监控 IO 统计信息

```bash
# 查看 IO 统计
iostat -x 1

# 输出示例
Device            r/s     w/s     rkB/s     wkB/s   await  svctm  %util
sda              10.5     5.2     42.0      20.8    8.5    2.1    3.3
```

### blktrace

**作用**：跟踪 Block 层 IO 请求

```bash
# 跟踪 IO
blktrace -d /dev/sda -o trace

# 分析结果
blkparse trace.blktrace.* > trace.txt
```

### perf

**作用**：性能分析工具

```bash
# 分析文件系统性能
perf record -e block:block_rq_issue -a sleep 10
perf report
```

## 三、文件系统 Benchmark

### fio

**作用**：灵活的 IO 测试工具

```bash
# 随机读测试
fio --name=randread --ioengine=libaio --iodepth=16 \
    --rw=randread --bs=4k --size=1G --filename=/data/test

# 顺序写测试
fio --name=seqwrite --ioengine=libaio --iodepth=16 \
    --rw=write --bs=1M --size=1G --filename=/data/test
```

### iozone

**作用**：文件系统性能测试

```bash
# 测试文件系统性能
iozone -a -s 1G -f /data/test
```

## 四、常见性能问题与优化

### 1. 随机 IO 性能差

**原因**：
- 碎片化
- 缓存未命中

**优化**：
- 使用 f2fs（闪存优化）
- 增加预读
- 优化应用 IO 模式

### 2. 写入性能差

**原因**：
- 同步写入
- 日志开销

**优化**：
- 使用异步写入
- 调整日志模式
- 优化回写参数

### 3. 内存压力

**原因**：
- 页缓存占用过多
- 脏页过多

**优化**：
- 调整脏页回写参数
- 限制页缓存大小
- 使用直接 IO
